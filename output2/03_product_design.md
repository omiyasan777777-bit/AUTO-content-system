# 商品設計書

生成日: 2026-05-13
コンセプト参照: output2/02_concept.md
コンセプト要約: 疲れ果てた普通の会社員が、日常経験だけを使って月1万円の副収入noteの土台を作る「省エネ積み上げメソッド」

## PART A: オファー設計

### 訴求4要素

```
■ 要素1: ベネフィット（得られる成果）
- メインベネフィット: 日常経験だけを使って有料note1本を公開し、月1万円の副収入の土台を完成させる
- サブベネフィット1: 「自分には実績がない」という思い込みが消え、発信への心理ブロックが外れる
- サブベネフィット2: 記事が資産として積み上がり、寝ている間にも収益が発生する仕組みができる
- サブベネフィット3: 疲れている日も、5分の作業だけで収益の土台が崩れない設計ができる
- 感情的ベネフィット: 「自分の力で稼いでいる」という自己効力感 ＋ 万が一の収入減への安心感

■ 要素2: 独自性（なぜこの商品なのか）
- 独自メソッド名: 省エネ積み上げメソッド
- 他商品にない要素:
  1. 「頑張る量」ではなく「設計の正しさ」で収益化できるという逆張り発想
  2. 月5万・月10万ではなく「月1万円」という圧倒的に現実的なゴール設定
  3. 日常経験→有料コンテンツ変換フレームワーク（「経験の切り取り7パターン」）
- 自分だけが語れる理由: 疲弊した状態で副業を始め、努力論なしで仕組みだけを整えた体験

■ 要素3: 実績・証拠
- 自分の実績: 月1万円→月3万円へのスケールを実現した設計実績
- 購入者の実績: 「副業初心者でも有料note1本を公開し初売上を達成」
- 具体的な数字:
  - 月1万円 = 1,000円の記事を10本販売
  - 1記事 = 執筆3〜5時間（1週末で完成）
  - 記事1本が半年後も売れ続ける資産型の仕組み

■ 要素4: 緊急性・限定性
- なぜ今買うべきか: 初期ローンチ価格（値上げ予定）＋今の消耗状態が続くほど副業を始めるタイミングが遅れる
- 限定要素: ローンチ価格は先着30名 or 72時間限定
- 見逃した場合の損失: 毎月給与だけに依存する状態が続く ＋ 「今月こそ始める」を繰り返して1年が過ぎる
```

### オファー構成

```
■ メイン商品
- コンテンツ本体: 約15,000〜20,000字・6章構成
- 形式: テキスト（note有料記事）

■ サポート設計
- 質問対応: あり（DM / コメント）
- 対応期間: 購入後30日間
- 対応回数: 3回まで
- コミュニティ: なし（シンプル設計 ＝ 本メソッドの世界観に一致）
- 添削・フィードバック: なし（自走できる設計のため不要）

■ 特典（3個）
- 特典1: 「経験の切り取り7パターンワークシート（穴埋め式）」
  ← 目的: 日常経験→コンテンツ変換を即実行できるようにする
  ← 単体想定価値: 2,000円

- 特典2: 「有料note1本の最小構成テンプレート（コピペ対応）」
  ← 目的: 「何を書けばいいかわからない」問題を完全解消
  ← 単体想定価値: 1,500円

- 特典3: 「月1万円達成チェックリスト（全20項目）」
  ← 目的: 自分の進捗が見える化され、続けやすくなる
  ← 単体想定価値: 500円

特典合計想定価値: 4,000円（本体価格 1,480円 を上回る）

■ 価格設計
- 本体価格: 1,480円（ローンチ価格）
- 値上げ計画: 1,480円 → 1,980円 → 2,480円（最終価格）
- 価格正当化ロジック:
  - 比較対象1: 副業スクール・コンサル = 5〜30万円 → それと比べて100分の1以下
  - 比較対象2: 競合の同テーマnote = 980〜2,980円 → 特典付きで同水準またはお得
  - 1日あたり換算: 約4円（365日で割ると）
  - 得られる成果の金銭的価値: 月1万円 × 12ヶ月 = 年間12万円の副収入土台

■ バックエンド導線
- 本商品の先に用意する上位商品:
  - 月3万円〜月5万円にスケールするnote副業実践ガイド（2,980〜4,980円）
  - 個別noteサポート（DM添削・フィードバック付き：5,000〜10,000円）
- 本商品内でのバックエンド導線の仕込み方:
  - まとめ章で「月3万円へのスケール設計図」として次のコンテンツを予告
  - 「このnoteだけで月1万の土台が作れる。次のnoteで月3万に拡張できる」という流れを提示
```

### 詳細ステップ

#### 大ステップ1: 思考リセット（副業の固定観念を壊す）

##### 小ステップ1-1: 「努力型副業」の思い込みを壊す
課題: 「副業=努力=自分には無理」という三段論法が頭の中にある
解決策: 設計が正しければ努力量に関係なく収益が出る事例と数字を示す

何をするか:
副業界隈の「毎日投稿」「根性」「頑張れ」文化がなぜ疲弊した人に刺さらないかを説明する。
「努力の量」ではなく「設計の正しさ」こそが収益を決める変数であることを、具体的な事例とともに解説する。

なぜやるか:
ここで思考リセットができないと、後のノウハウを読んでも「でも自分には無理」と感じてしまう。
全章の土台となる最重要パート。

コツ / 注意点:
- 努力を全否定するのではなく「設計なき努力が非効率」という表現にする
- 読者の過去の失敗体験を「あなたが悪いのではなく設計が悪かっただけ」と肯定する

成果物: 「自分にもできそう」という感覚が生まれ始める

---

##### 小ステップ1-2: 「実績ゼロ」という自己否定を外す
課題: 「特別な知識・実績がないと発信できない」という固定観念
解決策: 日常の経験は一次情報であり、専門家より読者に近い視点として価値がある

何をするか:
note上で実際に売れている記事が「普通の人の日常経験」であることを例示する。
「実績がない」ではなく「実績をまだ言語化していないだけ」というリフレーミングを提供する。

なぜやるか:
ターゲット読者の最大の心理ブロックがここにある。
解除できれば行動意欲が大きく変わる。

コツ / 注意点:
- 「実績がなくてもいい」という安易な励ましにならないようにする
- 「日常経験=一次情報=専門家より希少」という論理構造で納得させる

成果物: 「私の経験にも価値があるかもしれない」という気づき

---

##### 小ステップ1-3: 月1万円ゴールの解像度を上げる
課題: 「月1万円」が自分の手に届く数字として実感できない
解決策: 「1,000円×10本=月1万円」という最小モデルを可視化する

何をするか:
月1万円を「1,000円の記事が10本売れる」「1本/週ペースで10週間積み上げ」という具体的な数字に分解する。
週1本ペースの執筆スケジュール例を示し、「これなら無理ではない」と思わせる。

なぜやるか:
ゴールの解像度が低いと行動に踏み出せない。
「月1万円」を「今週1本書いて公開する」というレベルに落とし込む。

コツ / 注意点:
- 「月1万円を目指す」ではなく「月1万円の土台を設計する」という表現にする
- 最初から10本揃える必要はなく、1本ずつ積み上げれば良いことを強調

成果物: 月1万円への具体的なロードマップが頭に入っている状態

#### 大ステップ2: 経験棚卸し＆コンテンツ化

##### 小ステップ2-1: 自分の経験を棚卸しする
課題: 「自分に書けることが何も思いつかない」
解決策: 経験の切り取り7パターンワークシートで強制的にネタを発掘

何をするか:
7つの視点（①失敗談、②工夫・試行錯誤、③節約・時短、④感情の変化、⑤初心者の目線、⑥比較・検証、⑦「教えて」と言われること）から、自分の日常経験を書き出すワークを実施する。

なぜやるか:
「ネタがない」は思い込み。視点を与えれば誰でも10個以上のネタが出る。
このワークの完了が次のステップへの入力になる。

コツ / 注意点:
- 「すごい経験」ではなく「あたりまえの経験」を書き出すよう誘導する
- ワーク時間の目安を「15〜20分」と明示して心理的ハードルを下げる

成果物: 10個以上のネタリスト（ワークシート記入済み）

使うツール: 特典1「経験の切り取り7パターンワークシート」

---

##### 小ステップ2-2: ネタを「読者の悩み」に変換する
課題: 自分視点のネタを書いても読者に刺さらない
解決策: 「経験×読者の悩み」の掛け算フレームで価値に変換

何をするか:
棚卸しで出たネタを「読者の○○という悩みを解決するための△△」という形式に変換する。
変換前後の比較例を3〜5つ示し、読者が自分のネタで応用できるようにする。

なぜやるか:
自分視点のネタはそのままでは売れない。
「誰の何を解決するか」を明確にして初めて有料コンテンツになる。

コツ / 注意点:
- 変換は難しく考えず「これを知って私はどんな問題が解決できたか」だけ考えればいい
- 変換後の文章がそのままnoteのコンセプト文になることを伝える

成果物: 変換済みのコンセプト文1〜3個

##### 小ステップ2-3: 価格設定を決める
課題: いくらに設定すればいいかわからない
解決策: 文字数×独自性×ターゲット層の3軸マトリクスで価格を決定

何をするか:
市場相場（500〜2,980円帯）を整理したうえで、「文字数×独自性」の2軸マトリクスを使って価格を決定する方法を解説する。
初心者の場合は「まず980〜1,480円で出して実績を作る」戦略を推奨。

なぜやるか:
価格設定で悩んで公開が遅れるパターンを防ぐ。
「とにかく出してみる」行動を後押しする。

コツ / 注意点:
- 安すぎると価値が低く見られる（500円未満は避ける）
- 高すぎると初心者は売れない（3,000円超は実績が必要）
- 「最初は980〜1,480円」のレンジを具体的に示す

成果物: 価格（円）の確定

---

##### 小ステップ2-4: タイトルと導入文の設計
課題: タイトルが弱いと読まれない・買われない
解決策: 「ターゲット×ゴール×手段」のタイトル方程式＋3大地雷チェック

何をするか:
爆売れタイトルの方程式（[誰が]+[時間]+[結果]+[方法]）を使って、自分のnoteのタイトルを3〜5個作る。
3大地雷（抽象的・未来なし・ターゲット不在）のチェックリストで最終選定する。

なぜやるか:
どんなに内容が良くてもタイトルが弱ければクリックされない。
タイトル設計は購買率に最も直結するポイント。

コツ / 注意点:
- 「主婦が」「会社員が」などターゲット特化ワードを必ず入れる
- 数字（月○万円、○ステップ等）を必ず入れる

成果物: 確定タイトル1本

使うツール: 特典2「有料note1本の最小構成テンプレート」のタイトル欄

#### 大ステップ3: 最小構成note1本作成

##### 小ステップ3-1: 有料noteの構成を設計する
課題: 何をどんな順番で書けばいいかわからない
解決策: 最小構成テンプレート（6ブロック）に当てはめるだけ

何をするか:
有料noteの6ブロック構成（①導入・共感→②問題提起→③解決策の全体像→④詳細ノウハウ→⑤まとめ・行動促進→⑥次のステップへ）を解説する。
各ブロックの目標文字数と役割を明確に示す。

なぜやるか:
構成がないまま書き始めると途中で詰まる。
テンプレートがあれば「次に何を書くか」に悩まなくて済む。

コツ / 注意点:
- 全体の文字数は3,000〜8,000字が売れやすいゾーン
- 読者がスマホで読むことを想定し、1段落3〜4行を意識する

成果物: 有料noteの構成（6ブロック）が決まった状態

使うツール: 特典2「最小構成テンプレート」

##### 小ステップ3-2: 本文を書く（省エネ執筆法）
課題: 書き始めると手が止まる・書き疲れる
解決策: 1ブロック＝「課題→解決策→具体例→まとめ」の4行構造で書く

何をするか:
「省エネ4行構造」（①課題を1行で→②解決策を1行で→③具体例を2〜3行で→④ひと言まとめ1行）を紹介し、この繰り返しで全ブロックを埋める方法を解説する。
疲れた日は「1ブロックだけ書く」でも前進できることを伝える。

なぜやるか:
「全部書かなければ」という圧迫感が書けない原因になっている。
小分けにすることで疲れていても続けられる。

コツ / 注意点:
- 最初から完璧を目指さない（60点で公開して後から追記する戦略も有効）
- スマホのメモアプリで書き溜めたものをnoteに貼り付けるだけでもOK

成果物: 本文の初稿完成（3,000〜8,000字）

---

##### 小ステップ3-3: 無料部分（セールスレター）を書く
課題: 有料部分に誘導する無料部分が弱く、購入につながらない
解決策: 「共感→問題提起→解決策の予告→購入理由」の4段構成テンプレート

何をするか:
無料公開部分（＝購入を決めてもらうセールスレター相当）を4段構成テンプレートに沿って書く。
特に「共感パート」（読者の悩みを自分の言葉で代弁する）が最重要であることを強調する。

なぜやるか:
有料noteの購入率は無料部分の質でほぼ決まる。
ここが弱いといくら本文が良くても売れない。

コツ / 注意点:
- 共感パートは「読者の悩みをそのままの言葉で書く」（自分の言葉ではなく読者の声で）
- 「購入理由」は「今すぐ買う理由＋後悔しない保証」の2点を必ず入れる

成果物: 無料部分（セールスレター）完成（1,000〜2,000字）

##### 小ステップ3-4: 公開設定と初動施策
課題: 公開してもアクセスが0で誰にも見られない
解決策: 公開直後の初動3アクション（告知・ハッシュタグ・相互フォロー）

何をするか:
①noteのハッシュタグ3〜5個の正しい設定方法、②SNS（X/Instagram）での告知文テンプレート、③同テーマのnoteクリエイターとの相互フォロー・スキの戦略を解説する。

なぜやるか:
公開直後の初動が低いと検索アルゴリズムで埋もれる。
最初の1〜5件の購入が後続の購入を呼ぶ流れを作る。

コツ / 注意点:
- ハッシュタグは「副業」「note」など大きすぎるものより「note副業初心者」など中規模を狙う
- 告知はSNS × note の相互送客が鉄則

成果物: 有料note1本・公開済み

使うツール: 特典3「月1万円達成チェックリスト」

#### 大ステップ4: 資産型積み上げ設計

##### 小ステップ4-1: 記事をシリーズ化して資産を増やす
課題: 1本の記事だけでは月1万円に届かず、毎回ゼロから書くのが辛い
解決策: 「前編→後編」「入門→実践→応用」のシリーズ化で記事を有機的につなぐ

何をするか:
1本の記事から派生するシリーズ展開パターンを3つ紹介する（①深掘り型、②対象者変換型、③時系列型）。
シリーズ化すると読者が複数記事を購入し、月収が掛け算で増えることを示す。

なぜやるか:
月1万円は「1本を1万回売る」ではなく「10本を100回ずつ売る」が現実的。
記事の資産化・複線化が継続収益の鍵。

成果物: シリーズ計画（次の2〜3本のネタと順序が決まった状態）

##### 小ステップ4-2: 過去記事を再利用して負担ゼロで更新する
課題: 毎回新しいネタを考えるのが負担で続かなくなる
解決策: SNS投稿・日記・メモを「note素材」に再利用する3つの転用パターン

何をするか:
①X/Instagramの過去投稿をまとめてnoteに転用、②日記・メモを経験ネタに昇華、③過去のnoteをリライトして新記事化──3つのパターンを具体例付きで解説する。

なぜやるか:
「毎回ゼロから書く必要がある」という思い込みが続かない原因の一つ。
すでに書いたものを再利用することで執筆コストを大幅に削減できる。

コツ / 注意点:
- 「再利用＝コピペ」ではなく「素材から再構成」という視点で行う
- リライトは元の記事の視点を変えるだけで新鮮なコンテンツになる

成果物: 次のnoteの素材候補リスト

##### 小ステップ4-3: 月1万円達成チェックと次のステップ
課題: 「このまま続けていいのか」「自分は正しい方向に進んでいるか」が不安
解決策: 20項目チェックリストで現在地を把握し、次の行動を明確化する

何をするか:
月1万円の土台に必要な20の要素（記事本数、価格設定、タイトル、初動施策、シリーズ化進捗等）をチェックするリストを提供する。
チェックが足りない項目が「次にやること」になる設計。
月3万円へのスケール設計図（次のnoteの予告）を最後に提示する。

なぜやるか:
ゴールへの進捗が見えないと継続意欲が落ちる。
チェックリストで「あとこれだけ」と見える化することで行動が続く。

成果物: 現在地の把握 ＋ 次の行動リスト1〜3個

使うツール: 特典3「月1万円達成チェックリスト（全20項目）」

## 有料部分作成への引き継ぎメモ

- 導入章で特に強調すべきポイント:
  - 「設計が正しければ頑張らなくていい」という本書の核心を最初の500字で明言する
  - 読者が「自分もこれに当てはまる」と感じるペルソナ描写を冒頭に入れる

  - STEP 2-1（経験棚卸し）: 「やっぱり自分には何もない」と感じやすい → ワークの前に再度「日常経験=一次情報」を念押し
  - STEP 3-2（省エネ執筆）: 完璧主義が出て「全部書けないと公開できない」と詰まりやすい → 「60点で出す」戦略を明記

- ワーク / テンプレートが必要な箇所:
  - 第3章：経験棚卸しワークシート（特典1）
  - 第4章：最小構成テンプレート（特典2）+ 月1万円チェックリスト（特典3）

  - ○「月1万円の土台を設計する」　✗「月1万円を稼ぐ」

  - ○「省エネで動ける設計にする」　✗「楽して稼ぐ」

- コンテンツ融合法の実行方針:
