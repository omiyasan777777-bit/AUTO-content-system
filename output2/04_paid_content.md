## STEP 1：Codexとは何か ── 「指示するだけで動く自律型AI」

**Codexは、ChatGPTとは根本的に違うAIです。**

ChatGPTに聞くと「答えを返して」くれます。

> 「先週のアクセスログCSVから、エラーコードが500のものだけ抜き出して、
> 件数を集計して、レポート用のExcelに転記する」

これをChatGPTに頼むと——
「まず、Pythonを使ってCSVを読み込み、次にエラーコードでフィルタリングして……」

**ChatGPTは「何をすべきか教えてくれる家庭教師」。**

この違いが分かれば、Codexの本質は掴めています。

---

### Codexができること

Codexが得意なのは、「入力→処理→出力」がはっきりしている作業です。

- **ファイルの変換・整形**: CSVを別のフォーマットに変換する、不要な列を削除するなど
- **テキスト処理**: 大量のテキストから特定の情報を抽出する、フォーマットを統一するなど
- **ファイル操作**: ファイル名を一括変更する、特定の条件でファイルを仕分けるなど
- **レポート生成**: データを読み込んで、決まった形式のレポートを自動で作るなど
- **コード生成**: 「こういう処理をするスクリプトを作って」という依頼

コードが書けなくても大丈夫です。
Codexに「こういうことをしてほしい」と日本語で伝えれば、
必要なコードを自分で書いて、実行までしてくれます。

## STEP 2：なぜDockerが必要なのか ── 3つの理由

「Dockerを使わずに直接Codexを動かせばいいじゃないか」

これは正しい疑問です。
実際に、Dockerなしでも使える方法はあります。

でも僕が「Docker経由で使う」ことを強くすすめる理由が3つあります。

---

**理由① ローカル環境を汚さない**

Codexは作業をするためにPCのファイルにアクセスします。
Dockerなしだと、Codexが意図せずPCの重要なファイルを書き換えたり、
不要なファイルを大量に生成したりするリスクがあります。

Dockerを使うと、Codexは「箱の中」で作業します。
箱の外（あなたのPC）には、指定したフォルダ以外は一切触れません。

イメージとしては——
工事をするとき、養生シートで部屋を覆ってから作業するのと同じです。
作業が終わったら箱ごと片付けられる。PCは綺麗なまま。

---

**理由② セキュリティが担保される**

業務で使うということは、機密情報が入ったファイルを扱うこともあります。

Docker Sandbox という機能を使うと、Codexの「動ける範囲」をフォルダ単位で制限できます。
「このプロジェクトフォルダの中だけ触っていい」という設定ができるわけです。

会社のPCで使うときや、機密性の高いファイルを扱うときに安心して使えます。

---

**理由③ 環境の再現性がある**

「自分のPCでは動いたけど、別のPCでは動かない」——これはエンジニアの世界では「あるある」です。

Dockerを使うと、「動く環境」をそのまま保存・共有できます。
チームのメンバーに同じ環境を配ることもできます。
PCを買い替えても、Dockerさえあれば同じ環境をすぐ再現できます。

---

### まとめると…

| | Dockerなし | Dockerあり |
|---|---|---|
| PCへの影響 | ファイルが散らかる可能性あり | 箱の外には影響ゼロ |
| セキュリティ | アクセス制限が難しい | フォルダ単位で制限可能 |
| 環境の再現 | 難しい | コピーして即再現 |

「Dockerを完全に理解する必要はない」とお伝えしましたが、
この3つの理由だけは理解してください。

**「なぜDockerを使うのか」が分かっていると、セットアップの各手順に意味が生まれます。**

## STEP 1：Docker Desktop をインストールする

まずDockerを使えるようにします。

**▶ Mac の方はこちら**

1. ブラウザで「Docker Desktop download」と検索
2. 公式サイト（docker.com）の「Download for Mac」をクリック
3. お使いのMacに合わせて選択してください：
   - **Apple Chip（M1/M2/M3/M4）の方** → 「Apple Silicon」を選択
   - **Intel Macの方** → 「Intel Chip」を選択
4. ダウンロードした `.dmg` ファイルをダブルクリック
5. Dockerアイコンを「Applications」フォルダにドラッグ
6. アプリケーションフォルダからDocker.appを起動
7. 画面の指示に従ってインストールを完了させる
8. **PCを再起動する**（これを忘れると後で問題が出ます）

再起動後、画面右上のメニューバーに「クジラのアイコン 🐳」が表示されていればOKです。

---

**▶ Windows の方はこちら**

Windowsの方には、事前に1つだけお願いがあります。
**WSL2（Windows Subsystem for Linux 2）の有効化**が必要です。

「また難しい設定か……」と思ったかもしれませんが、大丈夫。2ステップで終わります。

**【WSL2の有効化】**

1. スタートメニューを右クリック →「Windows PowerShell（管理者）」を選択
2. 以下のコマンドをコピーして貼り付け、Enterキーを押す

```
wsl --install
```

3. インストールが完了したら、**PCを再起動する**
4. 再起動後に「Ubuntu」というウィンドウが自動で開いたら成功です（ユーザー名とパスワードを設定します）

**【Docker Desktopのインストール】**

1. ブラウザで「Docker Desktop download」と検索
2. 公式サイト（docker.com）の「Download for Windows」をクリック
3. ダウンロードした `.exe` ファイルをダブルクリックしてインストール
4. インストール完了後、**PCを再起動する**

再起動後、タスクバーに「クジラのアイコン 🐳」が表示されていればOKです。

---

### ✅ POINT：「アップデートしてください」と言われたら

古いバージョンだと、後で出てくる「Docker Sandbox」の設定が表示されないことがあります。

## STEP 2：Docker Sandbox を有効化する

Docker Desktopが起動できたら、次は「Docker Sandbox」を有効化します。

これはCodexが「PC全体」ではなく「指定した場所だけ」で作業するようにする設定です。
必ずONにしてください。

**手順：**

1. 画面右上（Mac）またはタスクバー（Windows）のクジラアイコンをクリック
2. 「Settings（設定）」を選択
3. 左メニューから「Features in development」または「Beta features」を選択
4. 「Docker Sandbox」または「Docker MicroVM Sandboxes」というトグルを探す
5. トグルをON（青色）にする
6. 「Apply & restart」ボタンをクリック
7. Docker Desktopが再起動するのを待つ（30秒〜1分程度）

再起動後、Docker Desktopが立ち上がっていれば完了です。

---

### ✅ POINT：「Docker Sandbox」が見つからない場合

Docker Desktopのバージョンによって、設定項目の場所や名前が異なることがあります。

見つからない場合は、Docker Desktopを最新バージョンにアップデートしてください。
アップデート後に再確認すると表示されることがほとんどです。

（Docker Sandboxは2025年11月リリースの Docker Desktop 4.50 以降に搭載された機能です）

## STEP 3：OpenAI APIキーを取得・設定する

Codexを動かすには「APIキー」が必要です。
APIキーとは「あなたのCodexアカウント」を識別するための「認証コード」のようなものです。

**手順：**

1. ブラウザで `platform.openai.com` を開く
2. OpenAIアカウントでログイン（アカウントがない場合は新規登録）
3. 画面左メニューの「API keys」をクリック
4. 「+ Create new secret key」ボタンをクリック
5. 名前欄に「codex-docker」など分かりやすい名前を入力
6. 「Create secret key」をクリック
7. 表示されたキー（`sk-` から始まる長い文字列）を**必ずこの場でコピー**する

> ⚠️ **重要**：このキーは**この画面を閉じると二度と表示されません**。
> 必ずコピーして、安全な場所（メモ帳やパスワードマネージャー）に保存してください。

**利用料金について：**

APIキーを使うと、使用量に応じて課金が発生します。
ただし最初の段階では非常に少額です。

Codexを試す範囲であれば、月額数百円〜数千円程度が目安です。
（OpenAIのダッシュボードでいつでも使用量を確認できます）

不安な方は、OpenAIのダッシュボードで「月額上限（Usage limit）」を設定しておくと安心です。

---

### ✅ POINT：課金設定の確認

`platform.openai.com` の「Settings → Billing」から、クレジットカードの登録と残高の確認ができます。
初めての場合は、5〜10ドル程度のクレジットを購入しておくと安心です。

`.env` ファイルの中身を以下のようにして保存します：

```
OPENAI_API_KEY=sk-xxxxxxxxxxxxxxxxxxxxxxxxxx
```

`sk-xxxx...` の部分を、STEP 3でコピーしたAPIキーに置き換えてください。

**▶ Windows の方**

5. ファイル名を `.env` に変更する（拡張子の `.txt` は消してください）

`.env` ファイルの中身を以下のようにして保存します：

```
OPENAI_API_KEY=sk-xxxxxxxxxxxxxxxxxxxxxxxxxx
```

### ✅ POINT：`.env` ファイルの注意点

- 等号（=）の前後にスペースを入れないでください（`OPENAI_API_KEY = sk-xxx` はNG）
- APIキーを貼り付けるときに、前後に余分なスペースが入らないよう確認してください
- このファイルは絶対にGitHubやSNSに公開しないでください（課金されます）

## STEP 2：Codexへの指示の書き方

「Codexに指示を書く」——これには少しだけコツがあります。

**良い指示と悪い指示の違いは「具体性」だけです。**

---

### 悪い指示の例

```
「CSVをきれいにして」
「このファイルを整理して」
「レポートを作って」
```

これでは Codex が「何を」「どのように」「どんな形式で」すればいいか分かりません。
人間でも困りますよね。

---

### 良い指示の3要素

Codexへの指示には、必ずこの3要素を含めてください。

**① 何を（対象）**：どのファイルを処理するか
**② どうする（処理）**：何をしてほしいか
**③ どんな形で（出力）**：結果をどんな形式・ファイル名で出してほしいか

---

### プロンプトテンプレート

以下のテンプレートに当てはめるだけで、Codexが動く指示が書けます。

```
【対象】
「[ファイル名またはフォルダ名]」を対象に作業してください。

【処理内容】
[具体的にやってほしいことを箇条書きで]
・
・
・

【出力形式】
結果を「[出力ファイル名]」として[形式（CSV/テキスト/Excel等）]で保存してください。
```

---

### テンプレートを使った例

STEP 1のタスクBをテンプレートに当てはめると：

```
【対象】
「sales_data.csv」を対象に作業してください。

【処理内容】
・「部門」列が「営業部」のものだけを抽出する
・「売上」列の合計を計算する

【出力形式】
結果を「sales_summary.csv」としてCSV形式で保存してください。
また、売上合計額を最終行に追記してください。
```

このように書くと、Codexが迷わず作業できます。

---

### ✅ POINT：ファイルパスは正確に

Codexが「ファイルが見つからない」というエラーを出す最大の原因は、
**ファイルのパス（場所の指定）が間違っている**ことです。

基本的には、`codex-workspace` フォルダ内に処理したいファイルを置いてから、
「`codex-workspace` の中の `[ファイル名]`」と指定するようにしてください。

## STEP 3：実行して、結果を確認する

指示が書けたら、いよいよ実行です。

1. Codex の Web UI または CLI を開く

3. 実行を開始する
4. 完了を待つ（タスクの規模によって数秒〜数分）

### 実行後の確認ポイント

Codexがタスクを完了したら、必ず以下を確認してください。

**① 指定したファイルが生成されているか**
`codex-workspace` フォルダを開いて、指定した出力ファイルが存在するか確認します。

**② 内容が正しいか**
ファイルを開いて、意図した内容になっているか確認します。
「なんとなく合ってそう」だけでなく、数件サンプルを目視確認してください。

**③ 元のファイルが破壊されていないか**
Docker Sandbox を有効にしていれば基本的に問題ありませんが、
念のため元ファイルが正常かも確認してください。

---

### もし結果が意図と違ったら

完璧な結果が1回目から出るとは限りません。
それは普通のことです。

結果が違ったら、**指示を修正して再実行するだけです。**

```
「出力されたファイルを確認しましたが、〇〇が期待と違います。
[具体的に何が違うか]
[どうなってほしいか]
修正して再実行してください。」
```

このように「何がどう違うか」を具体的に伝えれば、Codexはすぐに修正してくれます。

---

### ✅ POINT：よくあるエラーと対処法

**エラー①：「File not found」「ファイルが見つかりません」**
→ ファイルが `codex-workspace` フォルダに置かれていないか、
　 ファイル名の指定（大文字・小文字、拡張子）が間違っています。
　 ファイル名を正確にコピーして指示に貼り付けてください。

**エラー②：処理が完了しない（タイムアウト）**
→ タスクが大きすぎる可能性があります。
　 最初は「100行のCSV」など、小さなサイズのファイルで試してください。

**エラー③：結果ファイルが空になっている**
→ 条件指定が間違っている可能性があります。
　「[列名]が[値]のもの」という条件を、実際のCSVの列名・値と一致させてください。

**STEP 1：繰り返しをリストアップする**

```
あなたの繰り返し業務：
・
・
・
```

---

**STEP 2：「入力と出力」が明確かチェックする**

Codexに任せられる作業には、共通した特徴があります。

```
✅ Codexに任せやすい作業の特徴：
・「このファイルを渡したら、このファイルが出てくる」という流れが決まっている
・作業のルール（条件・フォーマット）が決まっている
・判断より処理の作業（「こっちの方が見栄えがいい」ではなく「このルール通りに変換する」）
```

リストアップした作業の中で、この特徴に当てはまるものに○をつけてください。

---

**STEP 3：プロンプトに落とし込む**

○をつけた作業を、前のページで紹介したテンプレートに当てはめて書いてみてください。

```

【処理内容】
・[ルール1]
・[ルール2]
・[ルール3]

【出力形式】
結果を「[出力ファイル名]」として[形式]で保存してください。
```

## ✅ よくあるトラブル解決集

**Q：指示通りにやってくれない**
A：指示が曖昧な可能性があります。「どの列を」「何の条件で」「どんな形式で」をより具体的に書いてみてください。

**Q：大きなファイルを処理しようとするとエラーになる**
A：ファイルサイズが大きすぎる場合があります。まず100〜200行程度の小さなサンプルで試してから、本番ファイルに適用してください。

**Q：Macでは動いたが、Windowsでは動かない**
A：ファイルパスの区切り文字の違いが原因のことがあります。プロンプト内のパス指定をWindows形式（`\`）ではなく、フォルダ名だけで指定してみてください。

**Q：毎回同じプロンプトを書くのが面倒**
A：よく使うプロンプトをテキストファイルに保存しておきましょう。「プロンプト集.txt」として `codex-workspace` フォルダに置いておくと、必要なときにコピーするだけで使えます。
