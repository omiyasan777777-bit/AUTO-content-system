# 第1章: なぜiPhoneだけでアメブロを「データ運営」できるのか

あなたは今、アメブロをどんな方法で運営していますか？

スマホアプリで記事を書いて、投稿ボタンを押す。それだけ、という方がほとんどではないでしょうか。

でも、本当はもっとやりたいことがあるはずです。

「どの記事がよく読まれているのか知りたい」
「人気記事のパターンを掴んで、次の記事に活かしたい」
「毎回同じ書き出しを手打ちするのが面倒だから、定型文をすぐ呼び出したい」
「PCがないと、本格的な運営ツールは使えないんだろうな……」

そう思って諦めていたこと、ありませんか？

この章では、「なぜiPhone1台でアメブロをデータ運営できるのか」をまず理解してもらいます。仕組みがわかると、実装の怖さが一気に消えます。


**まず知っておきたい「アクセス解析3ヶ月問題」**

アメブロには標準のアクセス解析機能がついています。1日ごとのアクセス数・訪問者数・ページビューが確認できる、あの画面です。

でも、ひとつ重大な仕様があります。

**アメブロのアクセス解析データは、3ヶ月分しか保存されません。**

3ヶ月を過ぎると、古いデータは自動的に消えます。見返すことができなくなります。

「半年前のあの記事、なんであんなにアクセスが来たんだろう？」
「去年の〇月に急にアクセスが伸びたのはなんでだったっけ？」

そう思っても、もうデータがない。過去を振り返れない。何が効いたのかわからないまま、また感覚で記事を書くしかない。

これがアメブロ運営者の多くが陥っている状態です。

でも今日から、あなたは違います。iPhoneのa-Shellを使って、毎日のデータを自動保存できるからです。


**a-Shell ── iPhoneに隠れていたLinuxの力**

a-Shellは、App Storeで無料でダウンロードできるiOSアプリです。

見た目は地味な黒い画面に白い文字が流れるだけ。でも、その中身はすごい。

a-Shellの中では**Python 3系が動きます**。しかも、`pip install` コマンドを使えばPythonのライブラリ（外部パッケージ）をインストールすることができます。

今回使うライブラリは3つです。

**requests**: Webページにアクセスしてデータを取得するライブラリ。Pythonの世界では最もよく使われるHTTP通信ライブラリのひとつです。

**feedparser**: RSSフィードを解析するライブラリ。アメブロを含むほとんどのブログにはRSSという仕組みがあり、feedparserを使えばそこから記事情報を取得できます。

**beautifulsoup4**: WebページのHTMLを読み解くライブラリ。取得したページの中から「必要なデータだけ」を取り出すために使います。

これら3つはすべて**ピュアPythonライブラリ**です。C言語などで書かれた部分を含まないため、a-ShellのPip環境でも問題なくインストールできます。


**アメブロとPythonをつなぐ橋 ── RSS**

アメブロには、各ブログのRSSフィードという仕組みが用意されています。

あなたのアメブロのRSSフィードのURLは、こういう形式です。

```
https://ameblo.jp/あなたのブログID/rss20.xml
```

このURLにアクセスすると、最新の記事情報（タイトル・投稿日時・URL・本文の一部）がXML形式で取得できます。ブラウザで開いてみると、テキストのような画面が表示されるはずです。

これを、Pythonの `feedparser` ライブラリで読み込む。するとPythonのコード上でデータとして扱えるようになります。記事の「いいね数」「コメント数」などの情報もRSSに含まれていることがあります。

今回のnoteでは、このRSSを使って「人気記事ランキング」を自動分析するスクリプトを作ります。


**iOS Shortcuts との連携でワンタップ化**

a-Shellは、iOSの「ショートカット（Shortcuts）」アプリと連携できます。

ショートカットは、iPhoneに最初から入っているアプリです。複数のアクションを「ショートカット」として登録しておき、ホーム画面のアイコンをタップするだけで実行できます。

a-Shellには「URLスキーム」という仕組みがあり、ショートカットから a-Shell のスクリプトを起動することができます。

たとえば、こういうことができます。

「ホーム画面の『アメブロ分析』アイコンをタップ」
→ a-Shellが起動してPythonスクリプトが実行される
→ アクセス解析データがCSVに保存される
→ 人気記事ランキングがクリップボードにコピーされる
→ アメブロアプリに移動して貼り付けるだけ

全部iPhoneで、全部ワンタップで完結します。

また、定型文管理のスクリプトでは、ショートカット経由でテキストをクリップボードに送ることができます。記事の書き出し・プロフィール文・定型のあいさつ文などを a-Shell のテキストファイルに保存しておき、ショートカットでワンタップ呼び出し。そのままアメブロアプリに貼り付けられます。


**この教材で実装する4つの機能**

今日中に、次の4つが動く状態にします。

機能1: **アクセス解析の自動保存**
毎日のアクセス数・訪問者数・PV数をCSVファイルに自動保存するスクリプト。3ヶ月で消えるアメブロのデータを永久保存します。

機能2: **RSS人気記事分析**
feedparserでアメブロのRSSを取得し、最新記事のタイトル・投稿日時・URLの一覧を出力するスクリプト。どんな記事が公開されているかをデータで確認できます。

機能3: **定型文テンプレート管理 + ショートカット連携**
よく使う文章をテキストファイルで管理し、ショートカット経由でクリップボードに送るスクリプト。記事を書くたびに同じ文章を手打ちする必要がなくなります。

機能4: **ホーム画面ワンタップ起動**
上記3つのスクリプトをショートカットアプリに登録し、ホーム画面のアイコンをタップするだけで起動できる状態にします。

では、準備から始めましょう。
# 第2章: a-Shellのセットアップ + Pythonライブラリ導入

# 第3章: 3ヶ月で消えるアクセス解析を自動保存する

この章では、アメブロのアクセス解析データを自動でCSVファイルに保存するスクリプトを作ります。

アメブロのアクセス解析は便利ですが、繰り返しになりますが3ヶ月分しか保存されません。今日以前のデータは、3ヶ月後には見ることができなくなります。

でも今日からは大丈夫です。毎日スクリプトを実行するだけで、データが永遠に積み上がっていきます。


**アメブロのアクセス解析ページについて**

アメブロの管理画面（スタッフブログや自分のアメブロ管理ページ）のアクセス解析を見てみましょう。日付ごとに、次の数値が表示されています。

- アクセス数（セッション数）
- 訪問者数（ユニークユーザー数）
- ページビュー数

これらをPythonのrequestsライブラリで取得し、CSVファイルに1行ずつ保存していきます。

**重要なポイント**: アメブロの管理画面はログインが必要なページです。Pythonでこれを取得するには、ログイン済みのセッション情報（Cookieと呼ばれるデータ）をPythonスクリプトに設定する必要があります。

この「Cookieをスクリプトに渡す」という作業が、この章の一番の山場です。難しくはありません。一緒に進めましょう。


**CookieをブラウザからコピーするSTEP**

まずiPhoneのSafariでアメブロにログインします。

次に、ログインした状態でSafariの開発者ツール的な機能……は残念ながらiPhoneのSafariには使いにくい面があります。そのため、**別の方法**でCookieを取得します。

最も簡単な方法は、次の手順です。

**手順1**: Safariでアメブロにログインした状態で、アクセス解析ページを開きます。URLは次のような形式になっています。

```
https://www.ameba.jp/ameba/analysetop/
```

**手順2**: このページのURLをそのままメモします。そして、ログイン後にアメブロがブラウザに保存する Cookie の値が必要です。

実は、一番手軽なのは **アメブロのアクセス解析APIに直接アクセスする** 方法です。アメブロの各ブログのアクセス数は、次のような公開に近い形で取得できる部分があります。

ただし、ここで正直にお伝えします。

アメブロの管理画面の細かいデータ（自分のブログのアクセス数の詳細）を取得するには、ログイン認証が必要です。これを自動化するには、ログイン後のCookieをスクリプトに設定する必要があります。

Cookie を取得する最も実用的な方法を説明します。


**SafariのCookieをa-Shellに渡す方法**

iPhoneのSafariでアメブロにログインした状態で、次の手順を行います。

**ショートカットアプリを使う方法**（最も手軽）

iOSのショートカットアプリで「JavaScriptでCookieを取得」するアクションを組み合わせる方法がありますが、セキュリティ上の理由でiOSはJavaScriptのCookieアクセスを制限しています。

そのため、**実用的な代替手段**を取ります。

アメブロのサービスには、ログイン後にアクセス解析データを取得できる非公式な仕組みもありますが、最も確実で長く使える方法は次の通りです。

**ベストプラクティス: 手動でCookieを一度だけ設定する**

PCまたはiPhoneのSafari開発者ツールが使える環境で一度Cookieを確認し、a-Shell内のテキストファイルに保存しておく。その後は自動でスクリプトが読み込む、という設計です。

iPhoneで Cookie を調べるには、Safari の設定から「Webインスペクタを有効にする」を設定し、MacのSafariのデベロッパーメニューを使う方法が正規の手順です。

でも「そんなMacない」という方のために、**もう1つのアプローチ**を紹介します。


**RSSとPythonを使ったアクセス傾向分析（Cookie不要版）**

実は、ログインなしで取得できるデータもあります。アメブロのRSSフィードです。

アメブロのRSSには次の情報が含まれています。

- 記事タイトル
- 投稿日時
- 記事のURL
- 本文の冒頭（summary）

RSSは Cookie 不要でアクセスできます。つまり、feedparser を使えばすぐに取得できます。

「でもRSSにはアクセス数が含まれていないんじゃないの？」と思った方、鋭いです。確かにRSSには「〇PV」という数値は入っていません。

ただし、こんな活用ができます。

- 更新頻度の記録（いつ何本投稿したかがわかる）
- 記事タイトルの蓄積（過去の全記事タイトルを一覧でCSVに保存できる）
- 投稿日時から曜日・時間帯の傾向分析

これらはアクセス解析の数値ではありませんが、「ブログ運営の記録」として価値があります。

では、この「Cookie不要版」のアクセス傾向記録スクリプトを作りましょう。


**スクリプト: アメブロ投稿記録をCSVに保存する**

a-Shellで `ameblo` フォルダに移動してから、次のスクリプトを作ります。

```
cd ameblo
edit save_posts.py
```

次のコードを入力します。

```python
import feedparser
import csv
import os
from datetime import datetime

# ========================================
# 設定: あなたのブログIDに変更してください
# ========================================
BLOG_ID = "your-blog-id"  # 例: "techblog-sample"

# ========================================
# RSSフィードのURL
# ========================================
RSS_URL = f"https://ameblo.jp/{BLOG_ID}/rss20.xml"

# ========================================
# 保存先のCSVファイル名
# ========================================
CSV_FILE = "ameblo_posts.csv"

def fetch_and_save():
    print(f"RSSフィードを取得中: {RSS_URL}")
    
    # feedparserでRSSを取得
    feed = feedparser.parse(RSS_URL)
    
    if not feed.entries:
        print("記事が取得できませんでした。ブログIDを確認してください。")
        return
    
    # CSVファイルが存在するか確認（初回はヘッダーを書く）
    file_exists = os.path.exists(CSV_FILE)
    
    # 既存データの記事URLを読み込む（重複を避けるため）
    existing_urls = set()
    if file_exists:
        with open(CSV_FILE, "r", encoding="utf-8") as f:
            reader = csv.DictReader(f)
            for row in reader:
                existing_urls.add(row["url"])
    
    # 新しい記事を追記する
    new_count = 0
    with open(CSV_FILE, "a", encoding="utf-8", newline="") as f:
        fieldnames = ["取得日時", "タイトル", "投稿日", "url"]
        writer = csv.DictWriter(f, fieldnames=fieldnames)
        
        # 初回のみヘッダーを書く
        if not file_exists:
            writer.writeheader()
        
        for entry in feed.entries:
            url = entry.get("link", "")
            
            # 既に保存済みの記事はスキップ
            if url in existing_urls:
                continue
            
            title = entry.get("title", "タイトルなし")
            published = entry.get("published", "日時不明")
            now = datetime.now().strftime("%Y-%m-%d %H:%M")
            
            writer.writerow({
                "取得日時": now,
                "タイトル": title,
                "投稿日": published,
                "url": url
            })
            new_count += 1
            print(f"保存: {title}")
    
    if new_count > 0:
        print(f"\n✅ {new_count}件の新しい記事を保存しました: {CSV_FILE}")
    else:
        print("\n✅ 新しい記事はありません（既存データは保持されています）")

if __name__ == "__main__":
    fetch_and_save()
```

入力できたら保存して実行します。

```
python3 save_posts.py
```

初回実行時は「RSSフィードを取得中」と表示され、取得できた記事タイトルが一覧で表示されます。`ameblo_posts.csv` というファイルが作られ、その中に記事一覧が保存されます。


**コードの意味を1行ずつ理解する**

コードを「コピペしたら動いた」で終わらせたくない。1行ずつ意味を理解することで、自分でカスタマイズできる力がつきます。

`import feedparser` → feedparserライブラリを読み込む。これがないと使えません。

`import csv` → CSVファイルを読み書きするためのPython標準機能。

`import os` → ファイルが存在するかどうか確認するための機能。

`from datetime import datetime` → 「今の日時」を取得するための機能。

`BLOG_ID = "your-blog-id"` → あなたのブログIDを入れる変数。例えばアメブロのURLが `https://ameblo.jp/hanako-blog/` なら `"hanako-blog"` が BLOG_ID です。

`RSS_URL = f"https://ameblo.jp/{BLOG_ID}/rss20.xml"` → f文字列（f-string）と呼ばれる書き方。`{BLOG_ID}` の部分に変数の値が埋め込まれます。

`feed = feedparser.parse(RSS_URL)` → feedparserでRSSを取得してパース（解析）します。結果が `feed` に入ります。

`feed.entries` → 取得した記事の一覧。`for entry in feed.entries:` でひとつずつ処理できます。

`os.path.exists(CSV_FILE)` → CSVファイルが存在するかチェック。`True`（あり）か `False`（なし）を返します。

`csv.DictWriter` → 辞書形式でCSVに書き込む機能。ヘッダー名を指定して扱いやすくなります。

`if url in existing_urls: continue` → すでに保存済みのURLはスキップする処理。毎日実行しても重複保存されないようにしています。

`datetime.now().strftime("%Y-%m-%d %H:%M")` → 今の日時を「2026-06-26 14:30」のような形式に変換します。


**毎日実行する習慣をつくる**

このスクリプトを毎日実行することで、データが積み上がっていきます。

RSSは最新記事しか取得できないため、毎日実行することで「全記事の投稿履歴」を少しずつ蓄積できます（初回実行時に直近の記事が、その後は新しい記事が追加されていきます）。

次の章で説明するショートカット連携を使えば、ホーム画面からワンタップで実行できます。

まずは今日、このスクリプトを動かしてみてください。`ameblo_posts.csv` が作られ、あなたのブログの記事一覧が保存されたら、それが第1歩の完成です。


**CSVファイルの確認方法**

保存されたCSVファイルを確認したいときは、a-Shell内でこのコマンドを使います。


# 第4章: RSSで人気記事を分析して次の記事テーマを決める

前の章では、RSSからブログの投稿記録をCSVに保存しました。

この章では、もう一歩進みます。

RSSから取得できる情報を使って「どんな記事が人気になっているか」を分析するスクリプトを作ります。

「でも、RSSにはアクセス数が含まれていないんじゃ？」と思いましたよね。その通りです。

ただ、RSSにはアクセス数以外にも「人気の傾向を読む手がかり」が含まれています。この章では、その手がかりを最大限に活用する方法を紹介します。


**RSSで分析できること**

アメブロのRSSフィード（`https://ameblo.jp/ブログID/rss20.xml`）には、次の情報が含まれています。

- 記事タイトル
- 投稿日時
- 記事URL
- 記事の本文冒頭（summary）

これだけでも、次のような分析ができます。

**タイトルのキーワード分析**: 最近の記事タイトルを一覧で見ると、どんなキーワードで書いてきたかが一目でわかります。「このキーワード多いな」「最近この話題書いてないな」という気づきが得られます。

**投稿パターンの確認**: 投稿日時を記録し続けることで「自分は週に何本投稿できているか」「何曜日に多く投稿しているか」というパターンがわかります。

**タイトル構造の振り返り**: 記事タイトルを一覧で見ると、「数字を入れた記事が多い」「疑問形のタイトルが少ない」など、パターンに気づけます。

さらに、**複数のブログのRSSを比較する**こともできます。同じジャンルで人気のアメブロユーザーのRSSを取得して、そちらのタイトル傾向と自分のブログを比べてみる。これが「競合分析」です。


**スクリプト: RSS人気記事分析ツール**

`ameblo` フォルダの中に新しいスクリプトを作ります。

```
edit analyze_rss.py
```

次のコードを入力します。

```python
import feedparser
from datetime import datetime
import re

# ========================================
# 設定
# ========================================
# 自分のブログID
MY_BLOG_ID = "your-blog-id"

# 比較したい参考ブログのIDリスト（空でもOK）
REFERENCE_BLOG_IDS = [
    # "popular-blog-1",
    # "popular-blog-2",
]

# ========================================
# 関数: RSSを取得して記事リストを返す
# ========================================
def fetch_rss(blog_id):
    url = f"https://ameblo.jp/{blog_id}/rss20.xml"
    print(f"取得中: {url}")
    feed = feedparser.parse(url)
    
    articles = []
    for entry in feed.entries:
        title = entry.get("title", "")
        published = entry.get("published", "")
        link = entry.get("link", "")
        summary = entry.get("summary", "")
        
        articles.append({
            "title": title,
            "published": published,
            "link": link,
            "summary": summary[:100]  # 本文冒頭100文字まで
        })
    
    return articles

# ========================================
# 関数: タイトルを分析する
# ========================================
def analyze_titles(articles, blog_id):
    print(f"\n{'='*50}")
    print(f"📊 ブログ分析: {blog_id}")
    print(f"{'='*50}")
    print(f"取得記事数: {len(articles)}件")
    
    if not articles:
        print("記事が取得できませんでした。")
        return
    
    # 記事一覧を表示
    print(f"\n【最新記事一覧】")
    for i, article in enumerate(articles, 1):
        print(f"{i}. {article['title']}")
        print(f"   投稿日: {article['published']}")
        print(f"   URL: {article['link']}")
        print()
    
    # タイトルに数字が入っているか
    titles_with_numbers = [a for a in articles if re.search(r'[0-9０-９一二三四五六七八九十百千万]', a['title'])]
    print(f"【数字入りタイトル】: {len(titles_with_numbers)}件 / {len(articles)}件")
    
    # タイトルに「？」が入っているか
    titles_with_question = [a for a in articles if '？' in a['title'] or '?' in a['title']]
    print(f"【疑問形タイトル】: {len(titles_with_question)}件 / {len(articles)}件")
    
    # タイトルが長いか短いか
    title_lengths = [len(a['title']) for a in articles]
    avg_length = sum(title_lengths) / len(title_lengths) if title_lengths else 0
    print(f"【平均タイトル文字数】: {avg_length:.1f}文字")
    
    # 最も短いタイトルと長いタイトル
    shortest = min(articles, key=lambda a: len(a['title']))
    longest = max(articles, key=lambda a: len(a['title']))
    print(f"【最短タイトル】: 「{shortest['title']}」({len(shortest['title'])}文字)")
    print(f"【最長タイトル】: 「{longest['title']}」({len(longest['title'])}文字)")

# ========================================
# メイン処理
# ========================================
def main():
    print("アメブロRSS分析ツール")
    print(f"実行日時: {datetime.now().strftime('%Y-%m-%d %H:%M')}")
    
    # 自分のブログを分析
    my_articles = fetch_rss(MY_BLOG_ID)
    analyze_titles(my_articles, MY_BLOG_ID)
    
    # 参考ブログがあれば分析
    for ref_id in REFERENCE_BLOG_IDS:
        ref_articles = fetch_rss(ref_id)
        analyze_titles(ref_articles, ref_id)
    
    print(f"\n{'='*50}")
    print("✅ 分析完了")
    print(f"{'='*50}")

if __name__ == "__main__":
    main()
```

保存して実行します。

```

```


**コードの意味を1行ずつ理解する**

`import re` → 正規表現（文字列のパターンマッチング）を使うための機能。タイトルに数字が含まれているか調べるために使います。

`REFERENCE_BLOG_IDS = []` → 比較したい参考ブログのIDリスト。空のリストなので、使いたいときだけIDを追加できます。`#` を外すと有効になります。

`feed = feedparser.parse(url)` → feedparserがRSSのURLにアクセスして、中身を取得・解析します。

`feed.entries` → 取得した記事のリスト。各 `entry`（エントリ）がひとつの記事に相当します。

`entry.get("title", "")` → その記事のタイトルを取得。取得できない場合は空文字を返します。

`summary[:100]` → summaryの先頭100文字だけを取得。長すぎる本文を切り詰めます。

`re.search(r'[0-9０-９一二三四五六七八九十百千万]', a['title'])` → タイトルの中に数字（半角・全角・漢数字）が含まれているかチェックします。

`sum(title_lengths) / len(title_lengths)` → リストの合計を要素数で割って平均を計算します。

`min(articles, key=lambda a: len(a['title']))` → 記事のリストの中から、タイトル文字数が最小のものを取得します。


**分析結果の使い方**

スクリプトを実行すると、次のような情報が出力されます。

「最新記事一覧」では、RSSに含まれる直近の記事（通常10〜20件）のタイトルと投稿日が一覧で見られます。これを見ながら「自分はどんな記事を書いてきたか」を振り返りましょう。

「数字入りタイトル」では、タイトルに数字を使っている記事の割合がわかります。一般に、数字入りタイトル（「〇〇する5つの方法」など）はクリックされやすいと言われています。少ない場合は意識して増やすのも手です。

「疑問形タイトル」では、「〇〇って何？」「〇〇はなぜ？」のような形のタイトルがどれくらいあるかわかります。読者の疑問に答える形のタイトルは検索流入を増やすと言われます。

「平均タイトル文字数」では、タイトルが長すぎないか・短すぎないかがわかります。アメブロのタイトルは30〜40文字が検索結果で表示される上限の目安とされています。

**参考ブログとの比較**

同じジャンルで人気のアメブロユーザーのブログIDを `REFERENCE_BLOG_IDS` に追加すれば、そのブログのタイトル傾向も分析できます。

「あの人はタイトルに必ず数字を入れているんだな」「疑問形タイトルが多い」「タイトルが短い」などの特徴が見えてきます。

自分のブログとの違いがわかれば、次に試すべきことが見えてきます。これが「データを見て次の一手を決める」という運営スタイルです。

**スクリプト: 定型文呼び出しツール**

次に、`templates.txt` から指定の定型文を取り出すスクリプトを作ります。

```
edit get_template.py
```

次のコードを入力します。

```python
import sys
import os

# ========================================
# 設定
# ========================================
TEMPLATE_FILE = "templates.txt"

def load_templates(filepath):
    """テンプレートファイルを読み込んで辞書形式で返す"""
    if not os.path.exists(filepath):
        print(f"テンプレートファイルが見つかりません: {filepath}")
        return {}
    
    templates = {}
    current_name = None
    current_content = []
    
    with open(filepath, "r", encoding="utf-8") as f:
        for line in f:
            line = line.rstrip("\n")
            
            # ===名前=== の行を検出
            if line.startswith("===") and line.endswith("==="):
                # 前のテンプレートを保存
                if current_name:
                    templates[current_name] = "\n".join(current_content).strip()
                
                # 新しいテンプレート名を設定
                current_name = line[3:-3]
                current_content = []
            else:
                if current_name:
                    current_content.append(line)
    
    # 最後のテンプレートを保存
    if current_name:
        templates[current_name] = "\n".join(current_content).strip()
    
    return templates

def main():
    templates = load_templates(TEMPLATE_FILE)
    
    if not templates:
        print("テンプレートがありません。templates.txt を確認してください。")
        return
    
    # 引数でテンプレート名が渡された場合
    if len(sys.argv) > 1:
        name = sys.argv[1]
        if name in templates:
            print(templates[name])
        else:
            print(f"テンプレート「{name}」が見つかりません。")
            print(f"利用可能: {', '.join(templates.keys())}")
        return
    
    # 引数なしの場合はメニューを表示
    print("📝 定型文テンプレート一覧")
    print("=" * 40)
    
    names = list(templates.keys())
    for i, name in enumerate(names, 1):
        # 内容の冒頭を表示
        preview = templates[name][:30].replace("\n", " ")
        print(f"{i}. 【{name}】 {preview}...")
    
    print("=" * 40)
    print("番号を入力してください（終了: q）: ", end="")
    
    choice = input().strip()
    
    if choice.lower() == "q":
        return
    
    try:
        index = int(choice) - 1
        if 0 <= index < len(names):
            selected_name = names[index]
            print(f"\n【{selected_name}】をコピー用に表示します:")
            print("-" * 40)
            print(templates[selected_name])
            print("-" * 40)
            print("\n↑ この文章をコピーしてアメブロアプリに貼り付けてください。")
        else:
            print("有効な番号を入力してください。")
    except ValueError:
        print("番号を入力してください。")

if __name__ == "__main__":
    main()
```

保存して実行してみましょう。

```
python3 get_template.py
```

テンプレート一覧が表示され、番号を選ぶとテキストが出力されます。


**コードの意味を1行ずつ理解する**

`import sys` → コマンドライン引数（スクリプト実行時に渡す値）を扱う機能。

`sys.argv` → スクリプト実行時に渡された引数のリスト。`python3 get_template.py あいさつ` と実行すると `sys.argv[1]` が `"あいさつ"` になります。

`line.startswith("===") and line.endswith("===")` → 行が `===` で始まり `===` で終わるかチェック。テンプレートの区切り行を判定しています。

`line[3:-3]` → 文字列のスライス。先頭から3文字と末尾から3文字を取り除きます。`===あいさつ===` が `あいさつ` になります。

`current_content.append(line)` → リストに行を追加していく。テンプレートの中身を1行ずつ集めています。

`"\n".join(current_content).strip()` → リストの要素を改行で結合し、前後の空白を除去します。

`len(sys.argv) > 1` → 引数が渡されているかチェック。`python3 get_template.py 締め` と実行すると True になります。

```python
# config.txt から読み込む
with open("config.txt", "r") as f:
    BLOG_ID = f.read().strip()
```

```python
with open("analysis_result.txt", "w", encoding="utf-8") as f:
```

**トラブルシューティング**

スクリプトがうまく動かないときの確認ポイントです。

**「ModuleNotFoundError: No module named 'feedparser'」というエラーが出る**
feedparserがインストールされていません。`pip install feedparser` を実行してください。

**「RSSフィードを取得中」で止まる・エラーが出る**
ブログIDが間違っている可能性があります。アメブロのURLを確認し、`ameblo.jp/` の後ろの部分が正しいブログIDかチェックしてください。

**「記事が取得できませんでした」と表示される**
プライベート設定のブログや、記事が0件のブログではRSSが取得できないことがあります。

**CSVファイルが開けない・文字化けする**
`open()` に `encoding="utf-8"` を指定しているため、UTF-8対応のアプリ（Numbers、Googleスプレッドシートなど）で開いてください。ExcelはBOM付きUTF-8が必要な場合があります。その場合は `encoding="utf-8-sig"` に変更してください。

**ショートカットでa-Shellが開かず止まる**
ショートカットアプリの設定で、a-Shellへのアクセス許可が有効になっているか確認してください。「設定」→「プライバシー」→「ショートカット」から確認できます。
