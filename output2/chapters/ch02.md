# 第2章: 三角運用の全体像──X×ブログ×ASPをどう連携させるか

---

## 2-1. 三角運用とは何か

「X（Twitter）とブログとASP、どう組み合わせて使うの？」

これを一言で言うと、こういう関係性です。

```
X（SNS）
   ↓ フォロワーをブログへ誘導
ブログ
   ↓ 読者をASP案件のページへ誘導
ASP（アフィリエイト）
   ↓ 成約で報酬発生
```

この3つを別々にやっても弱い。でも連携させると、弱点を補い合って仕組みが回り始めます。

**X（SNS）の役割：集客エンジン**

Xは「今」の情報を届けるSNSです。投稿した瞬間にフォロワーのタイムラインに流れます。フォロワーが増えれば増えるほど、ブログ記事を公開したときのアクセス数が増えます。

SEOが育つまでの「3〜6ヶ月の空白期間」を、Xの集客で埋めることができます。

**ブログの役割：収益エンジン**

ブログはGoogleで検索されることで読者を集め、ASP案件のリンクをクリックしてもらうことで報酬を生みます。「資産型コンテンツ」として一度書いた記事が何年も稼ぎ続けます。

Xとは違い、「検索意図がある読者」が来るので成約率（購入・申し込み率）が高い。

**ASPの役割：報酬エンジン**

A8.net・もしもアフィリエイトなどのASP（アフィリエイト・サービス・プロバイダ）に登録し、掲載する広告（案件）を選びます。読者が広告リンクをクリックして申し込みなどの成果を上げると、報酬が発生します。

ブログやXのASPアフィリエイトリンクを通じて、成約1件で数百円〜数万円の報酬になります。

## 2-2. なぜXから始めるのか

「ブログだけ書けばいいんじゃないの？」

そう思う方も多いと思います。でも、ブログ単体には致命的な弱点があります。

**ブログ単体の弱点**

新しいブログがGoogleで検索上位に表示されるまで、通常3〜6ヶ月かかります（第1章で説明したサンドボックス効果）。

つまり最初の3〜6ヶ月、いくら記事を書いても誰にも読まれない状態が続く。

これが3ヶ月で挫折する人を大量に生む原因のひとつです。

**XとブログをセットにするとXが穴を埋める**

Xのフォロワーが100人いれば、ブログ記事を書いてXで告知した日に100人に届く可能性があります。

ブログがSEOで強くなるまでの間、Xが即時集客の役割を担う。これが三角運用が強い理由です。

**主婦がXで発信しやすい理由**

「主婦の日常」「育児の本音」「副業の挑戦記録」は、Xで共感されやすいコンテンツです。同じ境遇の主婦がフォローしてくれやすく、フォロワーが増えやすい。

完璧な情報を発信しなくていい。「今日の育児の隙間30分でASPに登録した」という記録投稿でいい。その等身大の正直さが、共感を呼ぶのです。

## 2-3. 三角運用の1週間スケジュール（育児隙間版）

「週に何時間もかけられない」という方のために、1週間合計で2時間15分で三角運用を回すスケジュールを組みました。

**【基本の1週間】**

月曜日（10分）
→ Xに「パート換算投稿」を1本投稿
→ 例:「時給1,000円のパートで3時間働くより、転職サービスの紹介記事を1本書いた方が、半年後に多く稼げる可能性がある、という話をブログに書きました→（リンク）」

火曜日（30分）
→ ブログ記事のネタ決め・タイトル決め・見出し構成を作る
→ 子どものお昼寝中・幼稚園バスの待ち時間でOK

水曜日（10分）
→ Xに「正直レポート投稿」を1本
→ 例:「今週のブログ記事は1本しか書けなかった。でも育児の隙間で書けたのだから100点！」

木曜日（60分）
→ ブログ記事の本文を書く
→ 構成は昨日できているので「書くだけ」の状態にしておく

金曜日（10分）
→ Xに「有益ノウハウ投稿」を1本
→ 例:「主婦がアフィリエイトで稼ぎやすい案件TOP3（A8.netで見られます）」

土曜日（30分）
→ ブログ記事を公開・Xで告知
→ ASPリンクが正しく貼られているか確認

日曜日（15分）
→ 来週のネタを3つメモ・投稿予約を1本設定（余裕があれば）

**【週合計: 2時間15分】**

これが基本リズムです。

「木曜日に60分確保できない」という週は、15分×4回に分割してもOK。大事なのは1週間で「Xを3本・ブログを1本」のペースを守ることです。

## 2-4. 育児隙間別タスク設計

「今日は何分しかない」という日別の動き方です。

**隙間10分でできること**
- Xに投稿を1本（テンプレートから型を選んで穴埋めするだけ）
- ブログ記事のタイトル候補を3つメモ
- ASPで新しい案件を1つチェック

**隙間30分でできること**
- ブログ記事の見出し構成を作る（タイトル＋H2見出し3〜5本）
- X投稿を3本まとめて作って予約投稿にする
- 既存ブログ記事のASPリンクを1つ追加・修正する

**隙間60分でできること**
- ブログ記事を1本書ききる（1,000〜1,500字の記事なら60分で書ける）
- Xのプロフィールを更新してフォロワー数を確認
- 週次振り返り（記事数・フォロワー数・PV数のメモ）

今日の隙間が10分でも、30分でも、何かひとつ前に進める。
積み上げた分だけ、4ヶ月目以降の収益に直結します。
