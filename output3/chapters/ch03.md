# 第2章：Codexを実際に動かしてみる

「理屈はわかった。でも実際に使えるかどうか不安」

この不安を解消するのがこの章の目的です。

Codexを使ったことがない人が最初につまずくのは「何を入力すればいいかわからない」という壁です。この章を読めば、その壁がなくなります。

でもこの章を読めば...

Codexへの初めての指示が送れるようになります。指示の精度を上げるコツがわかります。うまくいかない時の対処法もわかります。

この章では以下を学びます。

- STEP 1：Codexへの初めての指示——コピペでOKなテンプレ
- STEP 2：Codexが正確に動く指示の書き方
- STEP 3：Codexがうまく動かない時の対処法

---

## STEP 1：Codexへの初めての指示——コピペでOKなテンプレ

### この段階での課題

「Codexに何を入力すればいいかわからない」という最初の壁を突破します。

### 解決策

まず、以下のテンプレートをそのままコピペして使ってみてください。

**【最初の指示テンプレート①：投稿アイデア生成】**

```
以下の条件でThreadsの投稿アイデアを10個作ってください。

テーマ：[あなたのアカウントのテーマを入力]
ターゲット：[あなたのターゲット読者を入力]
トーン：カジュアルで親しみやすい
文字数：各アイデアは150文字以内

各アイデアには「フック（最初の1文）」と「本文の方向性」を含めてください。
```

**【最初の指示テンプレート②：投稿文の生成】**

```
以下の条件でThreadsの投稿文を3パターン作ってください。

テーマ：[投稿したいテーマ]
メインメッセージ：[一番伝えたいこと]
文字数：各パターン200〜300文字
トーン：[カジュアル/プロフェッショナル/親しみやすい]

パターンAは感情訴求、パターンBは実績訴求、パターンCはノウハウ訴求で書いてください。
```

これだけです。最初はこれをコピペして、[ ] の中を自分の情報に書き換えるだけで始められます。

実際にやってみると「あ、本当に動いた」という感覚が生まれます。その感覚が次のステップへの自信になります。

✅ POINT

最初から完璧な指示を書こうとしなくていいです。まず動かしてみる。うまくいかなければ修正する。この繰り返しが、Codexを使いこなすための唯一の方法です。

## STEP 2：Codexが正確に動く指示の書き方

### この段階での課題

使い始めたはいいが、「なんか期待通りじゃない」という状況に陥る。

### 解決策

Codexの指示精度を上げる「3要素フレームワーク」を紹介します。

**3要素フレームワーク: 問題 × 目的 × 条件**

**問題（What）:** 何の問題を解決したいか
**目的（Why）:** なぜそれが必要か・どう使うか
**条件（How）:** 具体的な制約・フォーマット・数量

**悪い指示の例:**

```
投稿アイデアを作って
```

これでは「どんなテーマ？」「何個？」「どんなトーン？」が不明確すぎます。

**良い指示の例:**

```
【問題】Threadsで毎日投稿するためのネタ出しに時間がかかっています。
【目的】1週間分（7投稿）のアイデアをまとめて用意して、日々の作業を減らしたい。
【条件】
- テーマ：AI×副業
- ターゲット：副業初心者の会社員
- 1投稿あたり150〜250文字
- フック（最初の1文）は疑問形か驚きの事実から始める
- 各投稿に「学び」「行動促進」「共感」のどれかの目的を設定する
```

この違いは大きいです。「問題→目的→条件」の順番で書くだけで、Codexのアウトプットの質が劇的に上がります。

**よく使う条件の書き方例**

- 文字数：「○○文字以内」「○○〜○○文字」
- 数量：「○個生成して」「○パターン作って」
- フォーマット：「箇条書きで」「番号付きリストで」「表形式で」
- トーン：「カジュアルに」「プロっぽく」「友人に話しかけるように」
- 例示：「例えば〜のような感じで」

## STEP 3：Codexがうまく動かない時の対処法

### この段階での課題

「試してみたけど、なんかうまくいかない…」という詰まりを解消します。

### 解決策

Codexがうまく動かない時のパターンと対処法を整理します。

**パターン1：アウトプットが期待と違う**

対処法：「もっと〜にしてほしい」と追加指示する。「〇〇は良かったけど、△△をもっと具体的にして」のように部分的な修正指示が有効です。

**パターン2：指示が長すぎて消化しきれていない**

対処法：指示を分割する。一度に全部頼まずに、「まずAだけやって」→「次にBをやって」と段階的に指示する。

**パターン3：ツールのコードがエラーになる**

対処法：エラーメッセージをそのままCodexに貼り付けて「このエラーを直して」と伝える。Codexはエラーの修正も得意です。

**パターン4：Codexが「できません」と言う**

対処法：指示を変える。「〇〇を作って」ではなく「〇〇という問題を解決するにはどうすればいいか教えて、そして実行して」のように、問題から入る。

📝 ワーク

実際にCodexにアクセスして、テンプレート①「投稿アイデア生成」を使ってみてください。[ ] の中に自分のテーマとターゲットを入れて、実際に動かしてみることが大切です。
