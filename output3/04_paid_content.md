## 毎日投稿が続かない「本当の理由」

多くの人は、毎日投稿が続かない理由を「意志力の問題」だと思っている。

「もっと頑張れば続けられる」
「自分の根性が足りない」

でも、それは違う。

**毎日投稿が続かない理由は、仕組みがないからだ。**

考えてみてほしい。毎朝オフィスに出勤できるのは、なぜか。
「毎朝出勤しようと強く決意しているから」ではない。
電車の時間が決まっていて、会社のシフトがあって、仕組みがあるから出勤できる。

投稿も同じだ。毎朝「今日の投稿を考える」という作業が発生している限り、絶対に続かない。
**意志力に頼らず、システムに任せること**——それが解決策だ。

## 1-1. 3ツールの役割を5分で理解する

まず、全体像を把握しよう。

このシステムは3つのツールが分業している。
それぞれの役割を理解しておくことが、後のセットアップをスムーズにする鍵だ。

【図解プロンプト】3ツールの役割分担を表す横型フロー図。左から「Claude Code（工場：投稿文を大量生成する）」→「Buffer（倉庫＆配送センター：投稿をスケジュール管理して自動配信）」→「Threads（店頭：フォロワーに届く）」の3ボックスが矢印でつながる。各ボックスに役割説明テキストを添える。シンプルな色分け（青・緑・紫）。

### Claude Code の役割：「投稿の工場」

Claude Codeは、Anthropicが提供するAIコーディングアシスタントだ。
「コーディングアシスタント」という名前だが、コードを書くためだけに使う必要はない。

今回のシステムでは、Claude Codeを**「投稿文の自動生成工場」**として使う。

日本語でこう指示するだけでいい：
「私のキャラクター設定を踏まえて、Threadsの投稿を30本作ってください」

それだけで、30本の投稿ドラフトが生成される。
コードは一行も書かない。

### Buffer の役割：「投稿の自動配送センター」

Bufferは、SNS投稿のスケジュール管理ツールだ。

「何月何日の何時にこの投稿を出す」と登録しておけば、その時間になったら自動的にThreadsに投稿される。

手動でBufferに登録する方法と、API経由で自動登録する方法があるが、
このnoteでは**手動登録（コピペ）**を推奨する。理由は後述する。

### Threads の役割：「投稿の届け先」

Threadsは、Meta（Instagram）が提供するSNSプラットフォームだ。
BufferとThreadsを連携させることで、Bufferでスケジュール設定した投稿が自動的にThreadsに投稿される。

## 1-2. Claude Codeのセットアップとコスト管理

### どのプランを選ぶか

Claude Codeには主に2つの利用方法がある。

**① Claude Max プラン（月額定額）**
- 料金: 月100ドル〜（2026年3月時点）
- 特徴: 使い放題（一定の制限あり）
- 推奨: ヘビーユーザー向け

**② API従量課金**
- 料金: 使った分だけ課金
- 特徴: 初期費用なし、使わない月は0円
- 推奨: 月イチ2時間作業のライトユーザー向け

**このnoteのシステムには、API従量課金がおすすめだ。**

月1回の投稿生成作業（30本程度）なら、APIコストは通常100〜500円以内に収まる。
月100ドルのMaxプランより大幅に安い。

【図解プロンプト】Claude CodeのプランをMaxプランとAPI従量課金で比較する2カラム比較表のビジュアル。左カラム（Maxプラン）：月額100ドル、ヘビーユース向け、使い放題。右カラム（API従量課金）：月100〜500円、ライトユース向け、月イチ作業に最適。右カラムに「✅ おすすめ」のバッジ。

### コスト爆発を防ぐ設定

Claude Codeを使い始めて「気づいたら数千円溶けていた」という失敗談をよく見かける。
これを防ぐための設定を最初にやっておこう。

**① 月額上限を設定する**

Anthropicのコンソール（console.anthropic.com）にログインし、
「Billing」→「Usage Limits」から月額上限を設定する。

おすすめは**月1,000円（約7ドル）**。
投稿生成だけなら十分すぎる上限だ。上限に達したら自動的に停止される。

```
設定手順:
1. console.anthropic.com にアクセス
2. 左メニュー「Settings」→「Limits」
3. 「Monthly Spend Limit」を $7（約1,000円）に設定
4. 「Save」をクリック
```

**② 作業後は必ずClaude Codeを閉じる**

Claude Codeをバックグラウンドで起動したまま放置すると、意図せずAPIが呼ばれることがある。
作業が終わったら必ずウィンドウを閉じる習慣をつけよう。

## 2-1. 「自分らしい投稿」を量産するキャラ設定プロンプトの作り方

### AI感のある投稿と自分らしい投稿の違い

Claude Codeに「Threadsの投稿を30本作って」とだけ指示すると、
以下のような投稿が生成されることが多い。

```
❌ AI感の強い投稿例:
「AI活用で業務効率化が実現できます。
 以下の3つのポイントを押さえましょう：
 ①時間の節約 ②コスト削減 ③品質向上
 あなたも今日からAIを活用してみませんか？」
```

この投稿、読んで「面白い」と思うだろうか。
おそらく思わない。なぜか。

「誰が書いたかわからない」からだ。

SNSで反応が取れる投稿には共通点がある。
**「この人が言っているから面白い」という個性と文脈がある**ことだ。

だからこそ必要なのが「キャラ設定プロンプト」だ。
あなたの個性・価値観・話し方・バックグラウンドをClaude Codeに最初に伝えることで、
AIが「あなたらしい文体・視点・言葉選び」で投稿を生成するようになる。

【図解プロンプト】「キャラ設定なし」と「キャラ設定あり」の投稿の違いを左右で比較した対比図。左（キャラ設定なし）：無個性・教科書的な文体の投稿例。右（キャラ設定あり）：個性的・体験談ベース・話し言葉の投稿例。中央に「キャラ設定プロンプト」という矢印を挿入。

### キャラ設定プロンプトのテンプレート

以下のテンプレートをコピーして、【】内をあなた自身の情報に書き換えよう。

```
【キャラ設定プロンプト テンプレート】

あなたは私（[あなたの名前/ニックネーム]）の投稿アシスタントです。
以下のキャラクター設定を必ず守って投稿文を作成してください。

---

【プロフィール】
- 名前/ニックネーム: [例: まいちゃん / たろう]
- 職業/肩書き: [例: 副業ライター / 会社員×副業挑戦中]
- 主なテーマ: [例: AI活用 / Threads運用 / 副業]
- バックグラウンド: [例: コード知識ゼロから自動化を学んだ人間]

【口調・文体】
- 一人称: [例: 私 / 僕 / わたし]
- 話し方: [例: カジュアル / 親しみやすい / 体験談ベース]
- 文末: [例: 〜です。〜ました。/ 〜だよ。〜だった。]
- 絵文字: [例: 使わない / 1投稿に1〜2個]

【価値観・こだわり】
- 大切にしていること: [例: 再現性 / 正直さ / 非エンジニア目線]
- 言わないこと: [例: 「稼ぎましょう」系の煽り / 根性論]
- 得意なテーマ: [例: ツール活用 / 時短 / 失敗談から学んだこと]

【投稿の方向性】
- 投稿で伝えたいこと: [例: コードゼロでもAI自動化はできる]
- ターゲット読者: [例: Threads発信を諦めかけている非エンジニア]
- 読んだ後に感じてほしいこと: [例: 「自分にもできそう」という希望]

---

この設定を完全に理解しましたか？
理解できたら「キャラ設定完了。投稿生成の準備ができました」と返答してください。
```

**使い方:**
1. Claude Codeを起動する
2. 上記テンプレートの【】内を埋めて貼り付ける
3. Claude Codeが「キャラ設定完了」と返答したら設定完了
4. **この会話セッションを保持したまま**次のプロンプトに進む（セッションを閉じるとリセットされる）

## 2-2. Threadsでバズる5パターンの投稿型をClaude Codeに覚えさせる

キャラ設定が完了したら、次は「投稿の型」をClaude Codeに学習させる。

### なぜ「型」が必要か

毎回「いい感じの投稿を作って」と指示していると、投稿のテイストがバラバラになる。
あらかじめ「この型で作って」と指定することで、品質が安定し、バリエーションも出せる。

### Threadsでバズる5パターン

【図解プロンプト】5つの投稿型を並べた一覧カード図。各カードに投稿型の名前と特徴・使いどころを記載。①気づき型（水色）②ストーリー型（オレンジ）③リスト型（緑）④共感型（ピンク）⑤問いかけ型（黄色）。各カードに「反応率の目安」のバーを追加。

**① 気づき型**
自分が発見した知識・視点の変化を短く伝える型。
```
例:
「Threadsが続かない理由、わかった。

 毎日「何を投稿しよう」って考えてるから続かない。

 正解は毎日考えないこと。
 月1回まとめて作って、あとはタイマーに任せる。

 意志力は使わない。仕組みに任せる。」
```

**② ストーリー型**
自分の体験談・失敗談・変化を物語として伝える型。
```
例:
「2週間前の自分「Threads毎日投稿するぞ！」
 1週間後の自分「ネタが…ない…」
 今の自分「寝てる間に今日の投稿が出た」

 変わったのは根性じゃなくて仕組み。」
```

**③ リスト型**
「○○な○つのこと」形式で情報を整理して伝える型。
```
例:
「Threads自動化でやめたこと3つ

 ①毎日ネタを考えること
 ②投稿のタイミングを手動で管理すること
 ③「今日も投稿しなきゃ」と焦ること

 全部Bufferに任せた。」
```

**④ 共感型**
読者の「あるある」に共感し、同じ悩みを持つ仲間として語りかける型。
```
例:
「「SNSの自動化したい」って思ってGASの記事開いて、
 コードの羅列見て即閉じたことある人、

 正直に言うと私もそうだった。

 でも今はコード一行も書かずに自動化してる。
 方法があったんよ。」
```

**⑤ 問いかけ型**
読者に質問を投げかけ、コメントやエンゲージメントを引き出す型。
```
例:
「Threadsで一番しんどいのってどれ？

 A: 毎日ネタを考えること
 B: 投稿文を書くこと
 C: 継続すること
 D: 全部

 私はずっとDだったけど、今はABCを全部仕組みにした。」
```

### 投稿型をClaude Codeに覚えさせる

キャラ設定の直後に、以下のプロンプトを追加で送る。

```
【投稿型学習プロンプト】

以下の5つの投稿型を覚えてください。
今後の投稿生成では、これら5つの型をバランスよく使ってください。

① 気づき型: 自分が発見した知識・視点の変化を短く伝える
② ストーリー型: 自分の体験談・失敗談・変化を物語として伝える
③ リスト型: 「○○な○つのこと」形式で情報を整理して伝える
④ 共感型: 読者の「あるある」に共感し、仲間として語りかける
⑤ 問いかけ型: 読者に質問を投げかけ、コメントを引き出す

各投稿の特徴:
- 文字数: 100〜200文字（Threadsの最適文字数）
- 改行: こまめに入れて読みやすく
- 絵文字: キャラ設定に従う
- 売り込みなし: 価値提供に徹する

理解できたら「投稿型の学習完了。30本の生成が可能です」と返答してください。
```

## 2-3. 5分で30本の投稿ドラフトを生成する

### 30本生成プロンプト

## 3-1. 30本の投稿を最速でBufferにスケジュール登録する方法

### 手動登録 vs API登録——なぜ手動をすすめるか

「せっかくClaude Codeを使っているなら、Buffer APIで自動登録すればいいのでは？」
と思う人もいるだろう。

もちろんAPI経由で登録する方法もある。
ただし、このnoteでは**手動登録（コピペ）**を推奨する。理由は3つだ。

【図解プロンプト】「手動登録」と「API登録」を比較する2カラム表。手動登録の列：設定30分・コード不要・品質確認しながら登録できる。API登録の列：設定2〜3時間・APIキー設定・コード必要・一括登録可能。下部に「最初は手動からスタートが正解」というメッセージ。

**① 品質を目視確認しながら登録できる**
30本を1本ずつ登録する過程で、文章のミスや違和感を発見できる。
API登録だと一括投入になるため、後から気づいて修正する手間が増える。

**② 設定の複雑さがない**
Buffer APIの設定にはOAuth認証・アクセストークン取得・エンドポイント設定が必要。
非エンジニアには30分以上かかることが多く、エラー対処もある。
手動なら今すぐ始められる。

**③ 30本ならコピペでも20〜30分で完了する**
月1回の作業に30分使うことを考えると、API設定に2〜3時間かける必要はない。

## 4-3. Threads投稿にnote誘導CTAを自動挿入してnote収益を上げる

### Threadsの発信がnoteの売上につながらない理由

Threadsで投稿を続けていても、noteの売上が増えないという声をよく聞く。

原因はシンプルだ。
**「Threadsの投稿」と「noteの記事」がつながっていないからだ。**

Threadsの投稿でどんなに良い情報を発信していても、
フォロワーは「読んで満足」で終わってしまう。
noteへの誘導（CTA: Call to Action）がなければ、noteは売れない。

### note誘導CTAの設計

効果的なCTAには3つの条件がある。

```
① 投稿の内容と連動している
   ❌「noteはこちら」（脈絡がない）
   ✅「このワークフローの詳細手順はnoteで公開しています」（内容と連動）

② 読者に「次のステップ」を見せる
   ❌「良かったらnoteも見てください」（受け身）
   ✅「完全な設計図はnoteで無料公開中です」（具体的な価値）

③ 毎投稿につけない（5〜7本に1本）
   全投稿にCTAをつけると「売り込みアカウント」になる
   価値提供投稿5〜6本に1本だけCTAを挿入する
```

### Claude Codeで自動的にCTAを挿入させる

30本の中に自動的にCTAつき投稿を入れるよう、プロンプトに指示を追加する。

```
【CTA挿入プロンプト（30本生成プロンプトに追加）】

30本の投稿のうち、5本にnote誘導のCTAを自然に挿入してください。

CTAの条件:
- 投稿内容と関連したnoteの価値を1行で示す
- プレッシャーを感じさせない自然な誘導
- 投稿の最後に改行してから追加する

CTA例:
「このワークフローの全手順 → noteで公開しています（プロフリンクから）」
「詳しい設計図はnoteにまとめました」
「完全版（プロンプト全文）はnoteのリンクから」

注意:
- 5本以上にCTAをつけない
- CTAなしの25本は純粋な価値提供に徹する
```

【図解プロンプト】30本の投稿の構成比を表す円グラフ。25本（83%）を「価値提供投稿（CTA なし）」、5本（17%）を「CTA付き誘導投稿」として色分け。下部に「5:1の比率がnote収益と信頼構築を両立させる黄金比」というメモを添える。

### note × Threads 収益導線の全体像

【図解プロンプト】Threads → note の収益導線を表す漏斗（ファネル）図。上部（広）「Threadsフォロワー」→ 中部「CTA付き投稿でnoteへ誘導」→ 下部（狭）「note購入者」。各段階に「価値提供で信頼を積む」「5〜7本に1本のCTA」「noteで詳細・完全版を提供」のラベルを添える。

## 「仕組みを持つ人」と「持たない人」の差

今日から1ヶ月後を想像してほしい。

**仕組みを持たない人**: 毎日「今日の投稿どうしよう…」と悩み、結局投稿できない日が続き、3ヶ月後にThreadsを諦める。

**仕組みを持つあなた**: 毎日Threadsが自動で投稿され続け、フォロワーが少しずつ増え、3ヶ月後にはThreadsが「勝手に動く資産」になっている。

違いは才能でも根性でもない。**仕組みを持っているかどうか**だけだ。

## 次のステップ：仕組みをさらに進化させる

このnoteで構築したシステムをベースに、さらに発展させられる方向がいくつかある。

**① 投稿テーマの多様化**
月によってテーマのバランスを変えてみよう。
「先月は自動化テーマ中心だったから今月はストーリー系を増やす」など、
Claude Codeへの指示を月ごとに調整することで、フォロワーの飽きを防げる。

**② Buffer → Threads以外のSNSへの展開**
同じシステムをInstagram・X（Twitter）・Blueskyに展開できる。
Claude Codeで各SNS用に投稿テキストを最適化し、Bufferで一元管理する。
1回の作業で複数SNSを自動化できるようになる。

**③ note記事の内容からThreads投稿を自動生成する**
書いたnote記事をClaude Codeに読み込ませ、
「この記事のエッセンスをThreadsの投稿10本に変換して」と指示する。
noteとThreadsの内容が自動的に連動し、発信の一貫性が生まれる。

【図解プロンプト】「月イチ2時間システム」の発展形を表すロードマップ図。中心に「Claude Code × Buffer × Threads（本noteの内容）」を置き、周囲に「他SNSへの展開（Instagram/X/Bluesky）」「note記事→Threads自動変換」「投稿テーマの多様化」の3方向に矢印が伸びる放射状の図。

あなたに必要だったのは技術ではなく、「正しい設計図」だった。

月1回2時間。
