## PART A: オファー設計

### 訴求4要素

```
■ 要素1: ベネフィット（得られる成果）
- メインベネフィット: 1泊2日の大阪立ち飲み旅を、宿選び→エリア選び→夜の導線まで30分で設計し当日3軒以上まわり切れる
- サブベネフィット1: 大阪7エリアの特徴・向いている人・宿を取る駅名が比較表で一目でわかる
- サブベネフィット2: GPTsプロンプト10本コピペで、自分の旅タイプに合わせた計画が5分で出来上がる
- サブベネフィット3: 楽天トラベルの検索条件がエリア別にまとまっていて、宿探しの時間が10分の1になる
- 感情的ベネフィット: 「大阪の夜を自分で完全にデザインした」達成感と、酔ったあとも余裕で帰れる安心感

■ 要素2: 独自性（なぜこの商品なのか）
- 独自メソッド名: 逆算立ち飲み設計法
- 他商品にない要素:
  ① 「店リストを教えない」設計 → 情報が古くならない。旅タイプ別に何度でも再現できる
  ② 「宿を先に決める旅」ではなく「飲む街を先に決めて宿を後で取る」逆算の順序を体系化
  ③ GPTsプロンプト×大阪立ち飲み旅特化 → この組み合わせはnote市場に存在しない
- 自分だけが語れる理由: 大阪7エリアの立ち飲み文化と宿泊エリアの関係を実体験で検証済み

■ 要素3: 実績・証拠
- 自分の実績: 逆算設計法を実践し「酔ったあとに電車移動なし」の旅を複数回実現
- 具体的な数字: 7エリア比較 / モデルコース5本 / プロンプト10本 / 楽天トラベル検索条件エリア別整理

■ 要素4: 緊急性・限定性
- なぜ今買うべきか: 旅行前に読まないと、また「宿が遠くて3軒目で帰った」後悔が繰り返される
- 限定要素: ローンチ価格（最初の30名 or 7日間限定）で1,480円。以降は1,980円
- 見逃した場合の損失: 旅の計画に数時間かけても、宿の場所を間違えるだけで満足度が半減する
```

```
- 本商品内でのバックエンド導線の仕込み方:
  - 最終章で「他都市版を準備中」と案内
  - noteのマガジンフォローへの誘導
```

## 有料部分作成への引き継ぎメモ

- 導入章で特に強調すべきポイント:
  「立ち飲み旅の失敗は店のせいではなく宿のせい」という逆転発想の刷り込み。
  読者が「そうか、順番が逆だったんだ」と気づく体験を冒頭に作る。

- 第2章（エリア比較）の執筆方針:
  各エリアの記述は必ず同じフォーマットで揃える（どんな街か / 向いている人 / 初心者向け度 / ひとり飲み向け度 / 宿を取るなら何駅 / 楽天トラベルの検索駅名 / 注意点）。型の統一が読みやすさと「お金を払った感」を両立させる。

- 読者がつまずきやすいポイント:
  ① 「天満と梅田の違い」→ 地図イメージを言葉で説明する工夫が必要
  ② 「GPTsって有料版じゃないと使えない？」→ 無料版でも動くプロンプトに絞ること
  ③ 「楽天トラベルで駅名検索する方法」→ スクショがない分、手順を細かく書く

- ワーク / テンプレートが必要な箇所:
  - 第1章末: 旅タイプ判定チャート（穴埋め形式）
  - 第2章末: 7エリア比較表（テキスト一覧形式）
  - 第3章: 楽天トラベル検索条件一覧（エリア別リスト）
  - 第5章: プロンプト10本（コードブロック形式でコピペしやすく）

- 世界観に合った表現の具体例:
  - 「旅の設計図を手に入れる」
  - 「飲む街から逆算する」
  - 「酔ったあとに電車移動しない旅」
  - 「夜の導線を事前に引いておく」
  - 「宿が決まれば、あとは飲むだけ」

- コンテンツ融合法の実行方針:
  - ChatGPT旅行ガイド: プロンプトの丁寧な説明と「初心者が迷わない」配慮の仕方を参考に
  - 大阪飲み歩き体験記: 各エリアの「空気感・雰囲気・向いている人」の言語化スタイルを参考に
  - GPTsプロンプト集販売: プロンプトを商品の「核」として前面に出す構成と価格正当化ロジックを参考に
  - 自分の独自要素: 「逆算立ち飲み設計法」の思想 + 7エリア比較の型の統一 + 楽天トラベル検索条件の具体化
