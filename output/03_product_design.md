## PART A: オファー設計

### 訴求4要素

```
■ 要素1: ベネフィット（得られる成果）
- メインベネフィット: 先延ばしにしていた作業に、今日から24時間以内に1分で着手できる
- サブベネフィット1: 「やる気が出るのを待つ」ループから永久に抜け出せる
- サブベネフィット2: 25分集中できなくていい・集中できなくても着手できる仕組みがわかる
- サブベネフィット3: スマホのタイマーだけで完結する。道具・アプリ不要
- 感情的ベネフィット: 「自分はやればできる」という自己効力感の回復・罪悪感ゼロの毎日

■ 要素2: 独自性（なぜこの商品なのか）
- 独自メソッド名: 1分着手ポモドーロ
- 他商品にない要素: ポモドーロを「集中継続ツール」ではなく「着手専用スタートボタン」として完全再設計。25分集中を目標にしない唯一のポモドーロ活用術
- 自分だけが語れる理由: ポモドーロを集中ツールとして使って挫折した経験を持ち、「着手ツール」として再定義することで先延ばしが消えた実体験がある

■ 要素3: 実績・証拠
- 自分の実績: ポモドーロを着手ツールとして使い直してから、先延ばしにしていた作業に即日着手できるようになった
- 購入者の実績: 「1分でいいと思ったら急にハードルが下がって動けた」「集中しなくていいと知って気が楽になった」
- 具体的な数字: 着手までの時間が「やる気待ち数時間」→「1分以内」に短縮

■ 要素4: 緊急性・限定性
- なぜ今買うべきか: 先延ばしは1日単位で損失が積み上がる。今日着手できなかった作業は明日も着手できない確率が高い
- 限定要素: ローンチ価格は先着50名限定・期間限定
- 見逃した場合の損失: 毎日「今日もできなかった」という自己嫌悪が積み重なり、自己効力感がさらに低下する
```

### オファー構成

```
■ メイン商品
- コンテンツ本体: 約12,000〜15,000字・5章構成
- 形式: テキスト（note有料記事）

■ 特典（3個）
- 特典1: 1分着手ポモドーロ 即日実践シート（PDF）
  ← 目的: 読んですぐ使える穴埋め式ワークシート
  ← 単体想定価値: 500円

- 特典2: 先延ばしタスク別・着手ファーストワード集
  ← 目的: 「最初の1分に何をするか」を種類別に一覧化。迷いゼロで着手できる
  ← 単体想定価値: 300円

- 特典3: 「やる気待ちループ」脱出チェックリスト（10項目）
  ← 目的: 先延ばしのパターンを自己診断し、対処法を即特定する
  ← 単体想定価値: 300円

※ 特典合計想定価値: 1,100円 > 本体価格980円（特典で価値超え達成）

■ 価格設計
- 本体価格: 980円（最終定価）
- ローンチ価格: 680円（先着50名 or 発売後7日間）
- 値上げ計画: 680円 → 980円 → 1,480円
- 価格正当化ロジック:
  - 比較対象1: 自己啓発書（1,500〜2,000円）→ 読んで「わかる」だけ。この商品は「今日動ける」
  - 比較対象2: 生産性アプリ（月額500〜1,000円/月）→ 毎月かかる。これは一度買えば永久に使える
  - 1日あたり換算: 680円 ÷ 365日 = 約1.9円/日。コーヒー1杯以下
  - 得られる成果の金銭的価値: 先延ばしによる機会損失（仕事・副業・勉強）を考えると数万〜数十万円相当

■ バックエンド導線
- 本商品の先に用意する上位商品: 先延ばし完全攻略プログラム（習慣化・集中力・タスク管理を包括した上位コンテンツ）
- 本商品内での仕込み方: 終章で「着手できた後の継続設計」に触れ、上位コンテンツへの興味を喚起する
```

### 詳細ステップ

#### 大ステップ1: なぜ動けないのかを理解する

##### 小ステップ1-1: 先延ばしの正体を知る
```
課題: 「動けない自分＝意志が弱い」という誤った自己認識が行動を阻んでいる
解決策: 先延ばしは脳の防衛本能（不快回避）であると理解させ、自己否定を解除する

何をするか:
未知・困難・退屈なタスクに直面したとき、脳が「不快」を感じて回避行動を取る仕組みを説明。
先延ばしは「意志の弱さ」ではなく「脳が正常に機能している証拠」であることを解説する。

なぜやるか:
「自分はダメだ」という自己嫌悪が行動への最大の障壁になっている。
この認識を解除しないと、どんな手法を学んでも「また失敗した自分」になり続ける。

コツ / 注意点:
・「脳が悪い」という免罪符を与えすぎず、「仕組みを知れば変えられる」という希望につなぐ
・ここは感情的な共感が最優先。理論の詳細は短く済ませる

成果物: 「先延ばしは性格の問題ではない」という認識の転換が完了した状態
使うツール / リソース: 先延ばしの脳科学的説明（作業興奮・扁桃体・不快回避）
```

##### 小ステップ1-2: やる気待ちループの仕組みを把握する
```
課題: 「やる気が出たら始める」という発想そのものが問題だと気づいていない
解決策: やる気が「先に来る」という前提が根本的に誤りであることを構造的に示す

何をするか:
「やる気待ち→時間が過ぎる→罪悪感→さらにやる気が出なくなる」のループ構造を図解。
やる気は「行動の原因」ではなく「行動の結果」であることを示す。

なぜやるか:
このループの構造を理解しないと、1分着手ポモドーロの設計思想が腑に落ちない。
「なぜ着手だけでいいのか」の理論的裏付けになる章。

コツ / 注意点:
・ループの図解はシンプルに。複雑にしすぎない
・「あなたもこのループにいませんか？」という自己投影を促す問いかけを入れる

成果物: やる気待ちがなぜ機能しないかを言語化できる状態
使うツール / リソース: やる気待ちループの構造図
```

##### 小ステップ1-3: 自分の先延ばしパターンを特定する
```
課題: 先延ばしに悩んでいるが、自分のパターン（タイプ）を把握していない
解決策: 3タイプの診断チェックリストで自己理解を促す

何をするか:
「完璧主義型」「不快回避型」「優先順位迷子型」の3パターンを診断するチェックリストを提供。
自分のタイプに応じた1分着手ポモドーロの使い方の違いを簡単に示す。

なぜやるか:
「自分の問題が明確になった」という実感が購入者の満足度と行動率を高める。

コツ / 注意点:
・診断は5項目程度でシンプルに
・各タイプに「あなたは○型です」という明確なラベルを与える

成果物: 自分の先延ばしタイプが特定できた状態
使うツール / リソース: 3タイプ先延ばし診断チェックリスト（特典と連動）
```

#### 大ステップ2: やる気の仕組みを逆用する

##### 小ステップ2-1: 作業興奮の仕組みを理解する
```
課題: 「やる気がないと作業できない」という思い込みが行動を止めている
解決策: 作業興奮（動くことでやる気が生まれる脳の仕組み）を解説し、思い込みを逆転させる

何をするか:
「作業興奮」の仕組みを解説。脳内のドーパミン・側坐核の活性化が「動き始めること」で起きることを説明。
「やる気は動く前ではなく、動いた後に来る」という逆転の真実を提示する。

なぜやるか:
この章が本商品の理論的クライマックス。「だから1分着手でいい」という設計思想の根拠になる。

コツ / 注意点:
・脳科学の専門用語は最小限に。「脳が動き出しを検知してやる気スイッチが入る」程度の平易な表現で
・「やる気を待っていた自分は間違っていなかった。ただ、順番が逆だっただけ」という解放感を演出する

成果物: 「動いてからやる気が出る」という仕組みを体感的に理解できた状態
使うツール / リソース: 作業興奮のメカニズム図解
```

##### 小ステップ2-2: 着手の最小単位を設計する
```
課題: タスクが大きく見えて最初の一歩が踏み出せない
解決策: タスクを「1分でできる最小単位」に分解するワークを実施する

何をするか:
先延ばしにしているタスクを1つ取り上げ、「最初の1分にできること」だけに分解する練習をする。
例：「資料作成」→「フォルダを開く」「タイトルだけ書く」「箇条書きで見出しを3つ書く」

なぜやるか:
タスクのサイズが大きいまま着手しようとするから心理的ハードルが高くなる。
「最初の1分だけ」に目標を絞ることで、脳が感じる「不快」を最小化できる。

コツ / 注意点:
・「1分でできることか？」という問いで必ず検証させる
・読者に書き込みスペースを提供する（ワークシート形式）

成果物: 自分のタスクの「最初の1分」が具体的に言語化できた状態
使うツール / リソース: 最小着手単位ワークシート（特典と連動）
```

##### 小ステップ2-3: 「完璧にやろう」思考を手放す
```
課題: 「最初から完璧にやらないといけない」という思い込みが着手を妨げている
解決策: 最初の1分は「質」ではなく「着手という事実」だけが目的と定義する

何をするか:
完璧主義型先延ばしのメカニズムを解説。
「着手の目的は完璧なアウトプットではなく、脳のエンジンをかけること」という設計思想を伝える。

なぜやるか:
完璧主義は先延ばしの最大の共犯者。この思考を外さないと1分着手ポモドーロは機能しない。

コツ / 注意点:
・「汚くていい、遅くていい、最初の1分だけ動けばいい」というメッセージを繰り返す

成果物: 「完璧でなくても着手していい」という許可を自分に与えられた状態
使うツール / リソース: -
```

#### 大ステップ3: ポモドーロを着手ツールに再設計する

##### 小ステップ3-1: ポモドーロの誤解を解く
```
課題: ポモドーロ＝25分間集中し続けるツールだという誤解がある
解決策: ポモドーロの本来の設計思想（着手の強制・とにかく始めること）を解説する

何をするか:
一般的なポモドーロ活用法の問題点（「集中できない人が使えない」設計になっている）を指摘。
本来のポモドーロの価値は「25分集中」ではなく「タイマーをセットして強制的に着手させる仕組み」にあることを解説する。

なぜやるか:
「ポモドーロを試したけど続かなかった」という読者の失敗体験を再定義し、「使い方が間違っていただけ」という気づきを与える。

コツ / 注意点:
・「あなたの失敗はポモドーロのせいではない、使い方の問題だった」という解放メッセージを強く出す

成果物: ポモドーロを「着手ツール」として捉え直せた状態
使うツール / リソース: -
```

##### 小ステップ3-2: 1分着手ポモドーロを設定する
```
課題: 具体的にどうセットアップすればいいかわからない
解決策: スマホのタイマーだけで完結する手順を提供する

何をするか:
「1分タイマーをセットする→着手する→1分経ったら続けるか判断する」の3ステップを解説。
「25分を目標にしない・集中できなくてもOK・タイマーをセットすることが最初のゴール」という3原則を明示する。

なぜやるか:
シンプルな手順書があることで「今すぐ始められる」感が生まれる。

コツ / 注意点:
・アプリ不要・スマホのデフォルトタイマーでOKと明示する
・手順は3ステップ以内に収める

成果物: スマホタイマーを使った1分着手ポモドーロの設定が完了した状態
使うツール / リソース: 1分着手ポモドーロ セットアップシート（特典）
```

##### 小ステップ3-3: 最初の1分にやることを決める
```
課題: 「1分でいいと言われても、最初の1分に何をすればいいかわからない」
解決策: タスク別「着手ファーストワード」一覧を提供する

何をするか:
「資料作成」「勉強」「メール返信」「読書」「副業作業」など、タスク別に「最初の1分にやること（着手ファーストワード）」を一覧化。
読者は自分のタスクに該当するファーストワードを選んで実践するだけでいい。

なぜやるか:
「何をするか迷う」という思考コストが着手を妨げる。迷いをゼロにすることで即行動できる。

コツ / 注意点:
・ファーストワードは「動詞+対象物」の形式で統一（例:「フォルダを開く」「タイトルを書く」）
・10〜15種類程度のタスクをカバーする

成果物: 自分のタスクの「着手ファーストワード」が決まった状態
使うツール / リソース: 着手ファーストワード一覧（特典）
```

#### 大ステップ4: 今日から使える手順で実践する

##### 小ステップ4-1: 今日着手するタスクを1つ選ぶ
```
課題: 複数の先延ばしタスクがあり、どれから始めていいかわからない
解決策: 「最も先延ばし歴が長いタスク1つ」を選ぶ明確な基準を示す

何をするか:
タスク選択の基準（先延ばし歴が最も長いもの or 今週締め切りが来るもの）を提示。
「複数に手を出さない・今日は1つだけ」というルールを設定する。

なぜやるか:
複数タスクへの分散が着手を妨げる。1タスクに絞ることで意思決定コストをゼロにする。

コツ / 注意点:
・「今日のタスク」を紙に書かせるワークを入れる

成果物: 今日着手するタスクが1つに決まった状態
使うツール / リソース: 1タスク選択ワーク
```

##### 小ステップ4-2: 実際にタイマーをセットして着手する
```
課題: 「わかった」と「実際にやった」の間に最大の壁がある
解決策: 読者にこのページを読みながら今すぐタイマーをセットさせる

何をするか:
「ここで一度本を閉じて、今すぐスマホのタイマーを1分にセットしてください」という直接的な指示を入れる。
「タイマーをセットしたら着手フィニッシュです。集中できなくてもOK」という許可を与える。

なぜやるか:
知識は行動に変換されなければ意味がない。この章が「読んで終わり」を防ぐ最重要ポイント。

コツ / 注意点:
・「今すぐ」という言葉を強調
・「1分だけ。それだけでOK」という繰り返しで心理的ハードルを下げ続ける

成果物: 先延ばしにしていたタスクへの着手が実際に完了した状態
使うツール / リソース: -
```

##### 小ステップ4-3: 着手後の継続設計をつくる
```
課題: 1回着手できても翌日にはまた先延ばしに戻る
```
