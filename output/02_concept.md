## コンセプト4要素

### 誰が（ターゲット）
- 大カテゴリ: 大阪旅行を計画しているお酒好き
- 条件1: 立ち飲み・はしご酒に興味がある（ひとり旅 or 出張のついでに飲みたい）
- 条件2: どのエリアに泊まれば飲み歩きできるかが決められていない
- 条件3: 大阪の地理・飲み文化に詳しくない初〜中級者（地元民ではない）
- ペルソナ名: 「立ち飲み旅に憧れてる田中さん（32歳・会社員・ひとり旅経験あり）」

### どこまで（ゴール）
- ゴール: 1泊2日の大阪立ち飲み旅を「宿選び → エリア選び → 夜の導線」まで30分で設計完了し、当日ストレスゼロで飲み歩きできる
- 達成判定基準: 旅から帰ったとき「3軒以上まわれた・移動でしんどくなかった・また来たい」が全部成立している

### どうやって（手段・独自メソッド）
- 手段: 大阪7エリア比較表 + 楽天トラベル検索条件一覧 + GPTsプロンプト10本
- 独自メソッド名: **「逆算立ち飲み設計法」**
- 独自性:
  - 店リストではなく「エリアの回り方」を売る → 情報鮮度が落ちない
  - 「宿を先に決める旅」ではなく「飲む街を先に決めて宿を後で探す」逆算の順序
  - GPTsに条件を入力するだけで、自分の旅タイプに合った設計が30分で出来上がる

## コンセプト二層構造

### コアコンセプト（実利的価値）
> 大阪に詳しくないひとり旅勢・出張族が、GPTsと7エリア比較表を使って「飲む街→泊まる場所→夜の導線」を30分で設計し、1泊2日の立ち飲み旅を完走できるガイド

### 抽象的コンセプト（世界観）
> **「飲み旅の設計図」**

- ワーディングの方向性:
  - 「設計する」「組む」「決める」「逆算する」（能動的・構造的な言葉）
  - 「導線」「動線」「エリア」（設計者視点の言葉）
  - 「攻略」「解読」は使わない（重すぎる）
  - 「羅針盤」「地図」は補助的に使ってよい
- ビジュアルイメージ:
  - 居酒屋の赤提灯 × ミニマルな路地図
  - 「手書きの旅メモ」ではなく「構造化された設計書」の雰囲気

```
大阪旅行で飲み歩きしたいが、
どのエリアに泊まればいいかわからない人、
ひとり旅や出張ついでに立ち飲みを楽しみたいが

大阪7エリアの比較表と
GPTsプロンプト10本を使うことで

「飲む街 → 泊まる宿 → 夜の導線」を30分で設計し

大阪立ち飲み旅を1泊2日で完走できる

独自性:
「店を教えない。エリアの回り方と宿の取り方を設計図として売る」
= 逆算立ち飲み設計法

ストーリー:
店をリストアップして大阪旅行に行ったのに、
宿が遠くて3軒目で帰るハメになった。
「店より宿だ」と気づいてから、
飲む街を先に決めて宿を後で取る逆算順序に変えた。
それをGPTsに落とし込み、30分で設計できる形にまとめた。
```

## 仮想ストーリー

**【過去】**
大阪旅行の前に飲み屋を3時間かけてリストアップした。天満、裏なんば、新世界、全部調べた。でも取ったホテルは「安かったから」という理由で選んだ難波から30分の場所。1軒目は楽しかったが、2軒目のあとで「帰り電車大丈夫かな」が頭をよぎり始めた。3軒目は結局スキップした。調べた甲斐が半分しかなかった。

**【転機】**
「宿を飲み屋街に近い場所にすればよかっただけだ」という当たり前の気づきが転機だった。さらに、「どの街で飲むか」を最初に決めてから「その街の近くで宿を取る」という順番に変えるだけで、旅の設計が根本的に変わると気づいた。店を調べる前に、エリアと宿を決める。

**【変化】**
天満を選んで徒歩10分のホテルを取った旅では、5軒まわって翌朝スッキリ起きられた。GPTsに「ひとり旅・予算2.5万・初大阪・新幹線で帰る」と入力するだけで、宿・エリア・夜の回り方が5分で出てきた。旅の準備時間が10分の1になった。

**【使命】**
大阪の地理を知らなくても、この逆算設計の順番とGPTsプロンプトさえあれば、誰でも同じ旅ができる。飲み旅の失敗を防ぐ「設計図」を届けたい。

## 検証チェックリスト

- [x] ターゲットが「自分のことだ」と感じる具体性があるか → ひとり旅・出張・大阪不慣れの3条件で明確
- [x] ゴールに数字が含まれているか → 「30分で設計」「1泊2日完走」「3軒以上まわれた」
- [x] 手段に独自性・新規性があるか → 「店リストではなくエリア設計を売る」は競合にない
- [x] 網羅系になっているか → 7エリア比較 + モデルコース5本 + プロンプト10本
- [x] 競合と明確に差別化できているか → 「宿選びから逆算」は市場空白地帯

- [x] 抽象的コンセプト（世界観）が決まっているか → 「飲み旅の設計図」＋逆算立ち飲み設計法
- [x] 仮想ストーリーに感情的な説得力があるか → 「調べた甲斐が半分だった後悔」は刺さりやすい

## 商品設計への引き継ぎメモ

- このコンセプトから導かれる大まかなステップ案:
  1. 失敗パターンの提示（感情を刺す）
  2. エリア選びの基準を渡す（知識を与える）
  3. 目的別の宿泊エリア判定（実用価値を出す）
  4. モデルコースで完走イメージを作る（欲しいと思わせる）
  5. 楽天トラベル検索条件でアクション化（アフィ導線）
  6. GPTsプロンプトで再現性を担保（商品の核）

- 重点的に掘り下げるべきノウハウ:
  - 第2章（7エリア比較）: 型を揃えて読みやすく。初心者向け度×ひとり飲み度の2軸評価を必ず入れる
  - 第6章（プロンプト集）: 10本全て実用的なものに。「コピペで動く」が絶対条件

- 省略可能な要素:
  - 店名の列挙（情報鮮度リスクがある。「天満なら○○系の店が多い」という傾向表現に留める）
  - ChatGPTの基本的な使い方の説明（読者はある程度使える前提でよい）

- 世界観に合ったワーディング例:
  - 「設計する」「組む」「逆算する」「導線を引く」
  - 「旅のスタートは宿から」「飲む街から決めよ」
  - 「酔ったあとに電車に乗らない旅」

- コンテンツマージ元3商品の取り入れ方:
  1. ChatGPT旅行ガイド ← プロンプトの丁寧な説明と初心者への配慮の仕方
  2. 大阪飲み歩き体験記 ← 各エリアの「空気感・雰囲気」の言語化（数字ではなく感覚で伝える部分）
  3. GPTsプロンプト集販売 ← プロンプトをコンテンツの「核」として見せる構造と価格の正当化
