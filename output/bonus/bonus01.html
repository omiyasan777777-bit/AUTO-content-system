📋 指示書01：接続テスト（最初に必ず実行）
【作業の目的】
楽天市場の商品検索APIに接続できるかテストしてください。

【やってほしいこと】
- 楽天市場商品検索APIにリクエストを送ってください
- キーワード「ワイヤレスイヤホン」で商品を3件取得してください
- 各商品の「商品名」「価格」「アフィリエイトURL」を表示してください

【設定値】
- applicationId: "YOUR_APP_ID"
- affiliateId: "YOUR_AFFILIATE_ID"
- keyword: "ワイヤレスイヤホン"
- hits: 3

【制約・条件】
- 言語: Python
- requestsライブラリを使用してください
- 結果はターミナルに日本語で表示してください
- エラーが出た場合はエラー内容も日本語で表示してください
📋 指示書02：キーワード1件の商品検索
【作業の目的】
指定したキーワードで楽天市場の売れ筋商品を検索してください。

【やってほしいこと】

- キーワード: "ロボット掃除機"
- 売れ筋上位10件を取得してください
- 以下のデータを取得してください：商品名・価格・レビュー数・レビュー点数・アフィリエイトURL
- 結果をターミナルに番号付きで表示してください

【設定値】

- affiliateId: "YOUR_AFFILIATE_ID"
- keyword: "ロボット掃除機"
- hits: 10
- sort: "standard"（売れ筋順）

【制約・条件】
- 言語: Python
- requestsライブラリを使用してください
- 価格は「円」付きで表示してください
- レビュー点数は小数点1桁まで表示してください
📋 指示書03：複数キーワードの一括検索＋CSV保存
【作業の目的】
複数キーワードで楽天市場を一括検索し、結果をCSVファイルに保存してください。

【やってほしいこと】
- 以下のキーワードリストを順番に楽天市場商品検索APIで検索してください
  キーワードリスト: ["ロボット掃除機", "電動歯ブラシ", "プロテイン", "ヨガマット", "空気清浄機"]
- 各キーワードで売れ筋上位5件を取得してください
- 取得するデータ: キーワード・商品名・価格・レビュー数・レビュー点数・アフィリエイトURL・取得日
- 全商品データを「products_YYYYMMDD.csv」という名前で保存してください（YYYYMMDDは実行日の日付）

【設定値】
- applicationId: "YOUR_APP_ID"
- affiliateId: "YOUR_AFFILIATE_ID"
- hits: 5（1キーワードあたり）
- sort: "standard"（売れ筋順）

【制約・条件】
- 言語: Python
- CSVのエンコーディング: UTF-8-BOM（Excelで開けるように）
- 実行後に「○件の商品を取得してproducts_YYYYMMDD.csvに保存しました」と表示してください
- エラーが出たキーワードはスキップして次のキーワードに進んでください

- 各キーワードで売れ筋上位5件を取得してください
- 取得するデータ: キーワード・商品名・価格・レビュー数・レビュー点数・アフィリエイトURL・取得日
- 結果を「products_YYYYMMDD.csv」に保存してください

【設定値】

- affiliateId: "YOUR_AFFILIATE_ID"
- hits: 5
- sort: "standard"

【制約・条件】
- 言語: Python
- keywords.txtが存在しない場合は「keywords.txtが見つかりません」と表示して終了してください

📋 指示書05：高レビュー商品のフィルタリング
【作業の目的】
既存のCSVファイルから「レビュー数100件以上・評価4.0以上」の高品質商品だけを抽出してください。

【やってほしいこと】
- カレントディレクトリ内の最新の「products_*.csv」ファイルを自動検索して読み込んでください
- 以下の条件でフィルタリングしてください：
  条件1: レビュー数が100件以上
  条件2: レビュー点数が4.0以上
  条件3: 価格が1,000円以上50,000円以下
- フィルタリング後の商品リストを「filtered_products_YYYYMMDD.csv」として保存してください
- ターミナルに「全○件 → フィルタ後○件」と表示してください

【制約・条件】
- 言語: Python
- pandasライブラリを使用してください
- 価格列の「円」などの文字は数値変換時に除去してください
- フィルタリング後に0件だった場合は「条件に合う商品がありませんでした」と表示してください
📋 指示書06：note用記事文の自動生成（1商品版）
【作業の目的】
楽天商品データをもとに、noteに投稿する商品紹介記事文を生成してください。

【やってほしいこと】
以下の商品情報をもとに、note用の商品紹介記事文を生成してください。

商品情報:
- 商品名: （ここに商品名を入力）
- 価格: （ここに価格を入力）
- レビュー数: （ここにレビュー数を入力）
- レビュー点数: （ここに評価を入力）
- アフィリエイトURL: （ここにURLを入力）

生成する記事文の条件:
- 文字数: 300〜500文字
- 冒頭: 読者の悩みや欲求に共感する一文
- 商品の魅力を具体的に2〜3点紹介する
- 「こんな方におすすめ」のセクションを箇条書きで3項目
- レビュー数・評価を具体的な数字で言及する（信頼性の演出）
- 最後にアフィリエイトURLを自然に誘導する一文
- 記事の冒頭に「※本記事にはアフィリエイト広告のリンクが含まれています」を追加
- 絵文字を3〜5個使ってスマホで読みやすく

【制約・条件】
- 全て日本語で生成してください
- 事実に基づいた表現を使い、誇大表現は避けてください
- 結果をターミナルに表示してください
📋 指示書07：CSVから投稿文を一括生成＋テキスト保存
【作業の目的】
CSVファイルの全商品について、note用とX用の投稿文を一括生成してテキストファイルに保存してください。

【やってほしいこと】
- カレントディレクトリ内の最新の「products_*.csv」（またはfiltered_*.csv）を読み込んでください
- 各商品について以下の2種類の投稿文を生成してください：

  【A: note用記事文（300〜500文字）】
  ・冒頭に「※本記事にはアフィリエイト広告のリンクが含まれています」
  ・読者の悩みに共感する導入文
  ・商品の魅力を具体的に2〜3点
  ・「こんな方におすすめ」を箇条書き3項目
  ・レビュー数・評価を数字で言及
  ・最後にアフィリエイトURLを自然に誘導
  ・絵文字を3〜5個使用

  【B: X/Threads用短文（140文字以内）】
  ・商品名・価格・おすすめポイントを端的に
  ・ハッシュタグを3つ含める（#楽天 #楽天市場 #キーワード名）
  ・アフィリエイトURLを最後に配置

- 全投稿文を「posts_YYYYMMDD.txt」として保存してください
- 商品ごとに「===== 商品名 =====」という区切り線を入れてください

【制約・条件】
- 言語: Python
- 全て日本語で生成してください
- OpenAI APIを使用して投稿文を生成してください
  - model: "gpt-4o-mini"
  - openai_api_key: "YOUR_OPENAI_API_KEY"（OpenAIのAPIキーに書き換えてください）
- 実行後に「○件の投稿文を生成してposts_YYYYMMDD.txtに保存しました」と表示してください
📋 指示書08：自動実行メインスクリプト（一括実行）
【作業の目的】
楽天商品検索から投稿文生成までを一括で実行するメインスクリプトを作成してください。

【やってほしいこと】
以下の処理を順番に実行するPythonスクリプト「main.py」を作成してください。

処理の流れ:
1. keywords.txtからキーワードを読み込む
2. 楽天市場APIで各キーワードの売れ筋商品を検索（1キーワード5件）
3. レビュー数50件以上・評価3.8以上でフィルタリング
4. フィルタリング後の商品をproducts_YYYYMMDD.csvに保存
5. 各商品のnote用記事文とX用短文を生成
6. posts_YYYYMMDD.txtに保存
7. 実行ログをlog_YYYYMMDD.txtに記録

【設定値（スクリプトの先頭にまとめて定義）】
- RAKUTEN_APP_ID = "YOUR_APP_ID"
- RAKUTEN_AFFILIATE_ID = "YOUR_AFFILIATE_ID"
- OPENAI_API_KEY = "YOUR_OPENAI_API_KEY"
- HITS_PER_KEYWORD = 5
- MIN_REVIEW_COUNT = 50
- MIN_REVIEW_SCORE = 3.8

【制約・条件】
- 言語: Python
- エラーが発生した場合もスクリプト全体を止めず、ログに記録して続行してください
- 全処理完了後に「完了: 商品○件、投稿文○件を生成しました」と表示してください
- コメントを日本語で記述してください
📋 指示書09：Windowsタスクスケジューラー用バッチファイルの作成
【作業の目的】
Windowsタスクスケジューラーからmain.pyを実行するためのバッチファイルを作成してください。

【やってほしいこと】
- 以下の処理を行う「run_main.bat」を作成してください：
  1. Pythonの場所を自動検索して実行
  2. カレントディレクトリをスクリプトのあるフォルダに移動
  3. main.pyを実行
  4. 実行結果をbatch_log.txtに追記（実行日時付き）
  5. エラーが出た場合もbatch_log.txtに記録

- バッチファイルの保存先として想定するフォルダ:
  C:\Users\（ユーザー名）\楽天アフィリエイト\run_main.bat

【制約・条件】
- Windowsバッチファイル（.bat）形式で作成してください
- Pythonのパスが環境によって異なるため、`where python`で自動検索してください
- 文字コードはShift-JISで作成してください（Windowsのコマンドプロンプト対応）
- タスクスケジューラーへの登録手順もコメントとしてバッチファイル内に記載してください
📋 指示書10：実行状況レポートの生成
【作業の目的】
過去のログファイルと生成ファイルを集計して、実行状況レポートを生成してください。

【やってほしいこと】
- カレントディレクトリ内の以下のファイルを走査してください：
  - log_*.txt（実行ログ）
  - products_*.csv（商品データ）
  - posts_*.txt（投稿文）
- 以下の情報をレポートとして出力してください：
  - 自動実行の実行回数（ログファイル数）
  - 直近7日間の実行成功数・失敗数
  - 累計取得商品数
  - 累計生成投稿文数
  - 最後の実行日時
  - エラーが多いキーワードTOP3（ログから抽出）
- レポートをターミナルに表示し、「report_YYYYMMDD.txt」にも保存してください

【制約・条件】
- 言語: Python
- ファイルが1つもない場合は「まだ実行ログがありません」と表示してください
- 見やすい罫線付きのレポート形式で表示してください

💡 使い方のコツ

指示書のカスタマイズ方法

  • キーワードリストは自分の戦略に合わせて自由に変更できます
  • hitsの数を増やすと取得件数が増えます（楽天APIの上限は30件）
  • フィルタリング条件（レビュー数・評価・価格帯）は特典④のジャンル一覧を参考に調整してください
  • 指示書09と組み合わせて、タスクスケジューラーから指示書08のmain.pyを毎日自動実行するのが推奨構成です

エラーが出たときの対処

エラーメッセージをそのままCodexに貼り付けて「このエラーを修正してください」と指示するだけで、Codexが自動で修正コードを生成してくれます。
