A. Codexアプリのインストール・起動エラー

A-01：「WindowsによってPCが保護されました」が表示される

Windows
現象
Codexのインストーラーを実行すると青い警告画面が表示される
原因
Microsoft Defender SmartScreenが新しいソフトウェアをブロックしている
解決策
  1. 警告画面の「詳細情報」（下線付きのテキスト）をクリックする
  2. 「実行」ボタンが表示されたらクリックする
  3. インストールが再開される

A-02：「"Codex"は開発元を確認できないため開けません」が表示される

Mac
現象
Codexアプリをダブルクリックしても開けない
原因
macOSのGatekeeperがブロックしている
解決策
  1. Appleメニュー →「システム設定」を開く
  2. 「プライバシーとセキュリティ」を選択する
  3. 「セキュリティ」セクションに「Codexは…」という表示がある
  4. 「このまま開く」ボタンをクリックする
  5. 管理者パスワードを入力する

または：Finderでアプリケーションフォルダを開く → Codexアプリを右クリック（Controlキー+クリック）→「開く」を選択 →「開く」をクリック

A-03：Codexを起動するとAPIキーエラーが表示される

Windows / Mac
現象
起動直後に「APIキーが無効です」「APIキーを設定してください」と表示される
原因
APIキーが未設定、または誤ったキーが設定されている
解決策
  1. Codexアプリを開く
  2. 左サイドバーの「Settings」アイコンをクリックする
  3. 「API Key」欄の内容を削除する
  4. platform.openai.com でAPIキーを再確認してコピーする
  5. 「API Key」欄に貼り付けて「Save」をクリックする
B. APIキー・接続エラー

B-01：「Invalid API Key」エラー

Windows / Mac
エラーメッセージ
Invalid API Key / Authentication Error / 401 Unauthorized
原因と解決策
原因確認方法解決策
APIキーのコピーミスメモ帳でAPIキーを確認再度コピーして貼り直す
APIキーの前後にスペースAPIキーの先頭・末尾を確認スペースを削除する
APIキーが無効化されているplatform.openai.com で確認新しいAPIキーを発行する
支払い情報がないplatform.openai.com のBillingで確認クレジットカードを登録する

B-02：「Rate Limit Exceeded」エラー

Windows / Mac
エラーメッセージ
Rate limit exceeded / 429 Too Many Requests
原因
APIリクエストの回数・トークン数が上限を超えた
解決策
  • 1〜2分待ってから再実行する
  • gpt-4o-mini モデルを使用しているか確認する（上位モデルは制限が厳しい）
  • Codexへの指示書を1回1処理に絞り、1度に大量の商品を処理させない

B-03：「Insufficient quota」エラー

Windows / Mac
エラーメッセージ
You exceeded your current quota / Insufficient quota
原因
OpenAI APIの無料クレジットが切れた、または設定した上限に達した
解決策
  1. platform.openai.com の「Billing」を開く
  2. 現在の残高・使用量を確認する
  3. 残高が0の場合はクレジットカードを登録してチャージする
  4. 「Usage limits」で設定したHard limitを超えていないか確認する
C. 楽天APIエラー

C-01：「applicationId is required」エラー

Windows / Mac
エラーメッセージ
applicationId is required / パラメータエラー
原因
楽天のアプリIDが指示書に設定されていない、またはコピーミスがある
解決策
  1. webservice.rakuten.co.jp でアプリID（applicationId）を確認する
  2. 指示書の YOUR_APP_ID の部分が自分のアプリIDに書き換えられているか確認する
  3. アプリIDの前後にスペースや引用符が余分に入っていないか確認する
正しい形式 
applicationId: "1234567890abcdef"  ← ダブルクォートで囲まれている

C-02：アフィリエイトURLが通常のURLになっている

Windows / Mac
現象
affiliateUrl が https://item.rakuten.co.jp/... になっている（hb.afl.rakuten.co.jp ではない）
原因
アフィリエイトID（affiliateId）が設定されていない
解決策
  1. affiliate.rakuten.co.jp のダッシュボードでアフィリエイトIDを確認する
  2. アフィリエイトIDの形式は xxxx.xxxx.xxxx.xxxx の形式
  3. 指示書の YOUR_AFFILIATE_ID にアフィリエイトIDを設定する
  4. 再実行してURLが https://hb.afl.rakuten.co.jp/ で始まることを確認する

C-03：「No items found」/ 検索結果が0件になる

Windows / Mac
原因解決策
キーワードに特殊文字が含まれているキーワードをシンプルな日本語に変更する
在庫がない・販売停止のジャンル別のキーワードを試す
APIのgenresIdパラメータが誤っているgenresIdを削除してキーワード検索だけで試す

C-04：タイムアウトエラー / 接続エラー

Windows / Mac
エラーメッセージ
Connection timeout / Read timeout / ConnectionError
原因解決策
楽天APIが一時的に不安定5〜10分後に再実行する
VPNを使用しているVPNをオフにして試す
ファイアウォールがブロックしているファイアウォールの設定を確認する
ネットワークが不安定有線接続またはWi-Fiを変えて試す
D. 投稿文生成エラー（OpenAI API）

D-01：日本語が文字化けする

Windows / Mac
現象
生成された投稿文が文字化けしている
Windows の解決策 
# コード内でこのように修正
with open('posts.txt', 'w', encoding='utf-8-bom') as f:
Mac の解決策 
export LANG=ja_JP.UTF-8
Codexへの修正依頼
「生成されたテキストファイルが文字化けしています。UTF-8-BOMで保存するよう修正してください」

D-02：生成された投稿文の品質が低い・自然でない

Windows / Mac
現象
投稿文が機械的・不自然・繰り返しが多い
解決策：指示書の「制約・条件」に追記する内容 
【追加する条件（指示書の「制約・条件」に追記）】
- 毎回異なる書き出しパターンを使ってください
- 「注目の商品」「おすすめの商品」という定型文を使わないでください
- 商品の具体的な特徴（素材・重量・対応年齢など）を必ず1つ以上入れてください
- 読者に語りかける自然な口語体で書いてください
E. CSVファイル・ファイル保存エラー

E-01：CSVファイルが保存されない

Windows / Mac
原因確認方法解決策
保存先フォルダが存在しないフォルダを確認フォルダを作成してから実行
書き込み権限がない管理者権限で実行保存先を別のフォルダに変更
ファイルパスに日本語が含まれているパスを確認保存先を英語パスに変更

⚠️ Windowsユーザーへの注意： 保存先パスに「ユーザー名が日本語」の場合にエラーが起きることがあります。C:\Users\user01\楽天アフィリエイト\ のように英数字のユーザー名フォルダを使うか、C:\rakuten\ など別の場所に作業フォルダを作成することを推奨します。

E-02：ExcelでCSVを開くと文字化けする

Windows
現象
CSVファイルをダブルクリックしてExcelで開くと日本語が文字化けしている
原因
CSVがUTF-8で保存されているが、ExcelはShift-JISを期待している
解決策A（推奨）：CSVをUTF-8-BOMで保存する 
df.to_csv('products.csv', encoding='utf-8-bom', index=False)
解決策B
Excelで「データ」タブ→「テキストから取り込み」でUTF-8を指定して開く
Codexへの修正依頼
「CSVをUTF-8-BOMエンコーディングで保存するよう修正してください」
F. 自動実行エラー（タスクスケジューラー/launchd）

F-01（Windows）：タスクが「完了」するがファイルが生成されない

Windows
現象
タスクスケジューラーで「最後の実行結果：操作が正常に完了しました」と表示されるが、CSVが生成されていない
原因と解決策

1. バッチファイルのパスが間違っている

  • run_main.bat をメモ帳で開いて main.py のフルパスを確認する
  • パスに日本語や特殊文字が含まれていないか確認する

2. Pythonがパスに登録されていない

  • コマンドプロンプトで python --version を実行して確認する
  • バッチファイル内でPythonのフルパスを指定する：
"C:\Users\ユーザー名\AppData\Local\Programs\Python\Python3xx\python.exe" main.py

3. 作業ディレクトリが設定されていない

  • タスクのプロパティ→「操作」→「開始」欄に作業フォルダのパスを入力する

F-02（Windows）：タスクスケジューラーでエラーコードが表示される

Windows
エラーコード意味解決策
0x1スクリプトがエラーで終了したlog_*.txtを確認してPythonエラーを修正する
0x2ファイルが見つからないバッチファイルのパスを確認する
0xC000013Aユーザーがログオフしているタスクのプロパティで「ユーザーがログオンしているかどうかにかかわらず実行する」を選択する
0x41301タスクが現在実行中前回の実行が完了していない。スクリプトの実行時間を確認する

F-03（Mac）：launchdが実行されない

Mac
確認コマンド 
# launchdへの登録確認
launchctl list | grep rakuten

# 手動実行テスト
launchctl start com.rakutenaffiliate.daily

# ログの確認
cat ~/楽天アフィリエイト/launchd_out.log
cat ~/楽天アフィリエイト/launchd_err.log
よくある原因と解決策
原因解決策
plistが正しく登録されていないlaunchctl load ~/Library/LaunchAgents/com.rakutenaffiliate.daily.plist を再実行
plistのパスが間違っているplistファイルをメモ帳で開いてパスを確認・修正する
plistの権限が適切でないchmod 644 ~/Library/LaunchAgents/com.rakutenaffiliate.daily.plist を実行
Macがスリープ中スリープ設定を変更するか、実行時刻を変更する
G. Pythonライブラリ・環境エラー

G-01：「Module not found」エラー

Windows / Mac
エラーメッセージ
ModuleNotFoundError: No module named 'requests'（requestsの部分は他のライブラリ名の場合もあり）
解決策（Windows） 
pip install requests openai pandas
解決策（Mac） 
pip3 install requests openai pandas

G-02（Windows）：pip コマンドが認識されない

Windows
エラーメッセージ
'pip' は内部コマンド...として認識されていません
解決策
  1. コマンドプロンプトで python --version を実行してPythonがインストールされているか確認
  2. Pythonがインストールされていない場合：python.org からPython 3をダウンロードしてインストール
  3. インストール時に「Add Python to PATH」にチェックを入れることが重要
  4. インストール後にコマンドプロンプトを再起動して再試行

G-03：Pythonスクリプトの文字化け（Windows）

Windows
現象
スクリプト実行時に「SyntaxError: Non-UTF-8 code」エラーが出る
解決策：スクリプトの先頭に以下を追加する 
# -*- coding: utf-8 -*-

🆘 解決できない場合

上記で解決できない場合は、以下の情報をCodexに貼り付けてください：

【エラー報告テンプレート】
1. 発生したエラーメッセージ（全文をコピー）:
2. 使用OS: Windows / Mac（バージョン: ）
3. Pythonバージョン（python --version の結果）:
4. 実行したコマンドまたは指示書番号:
5. エラーが発生した状況（何をしたときに起きたか）:

Codexへの依頼文：「以下のエラーが発生しました。原因と修正方法を教えてください。[エラー内容を貼り付け]」

Codexがエラーの原因を分析して修正コードを生成してくれます。
