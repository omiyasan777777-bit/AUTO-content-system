## なぜ「ヘタウマ絵+ナレーション」が月収61万円を生み出すのか

設計図を組み立てる前に、まず「なぜこれが機能するのか」を理解してほしい。

理由を知らずに手順だけ真似しても、ニッチを間違えたとき修正できないからだ。

**理由1：ヘタウマ絵は「人間らしさ」を演出する**

AIが量産したような完璧な映像より、少し荒削りな絵の方が視聴者の共感を得やすい。

「この人が手で描いたんだ」という親近感が生まれ、エンゲージメントが上がる。

完成度を上げようとするほど、逆に「企業の広告っぽさ」が出て離脱される。

これは、落書き系チャンネルが多数のジャンルで再現されている心理的な現象だ。

**理由2：画像切り替え2〜3秒の法則**

1枚の画像の表示時間が2〜3秒以下の動画は、視聴維持率が上がる傾向がある。

眼が次の刺激を求めて前に進む。
飽きる前に次の画像が来る。

動画が10分でも最後まで見てもらいやすい。

1本の動画に画像が100枚あれば、それだけで「次の絵が見たい」という牽引力が生まれる。

**理由3：ナレーション主導のストーリー性**

視覚情報（絵）よりも聴覚情報（ナレーション）がメインになる構成。

スマホを伏せたまま音声だけで「聴けるコンテンツ」になる。

これはポッドキャストに近い視聴体験で、通勤・家事中のながら聴きに対応している。

再生数が伸びる理由は、こういう積み重ねだ。

## ヘタウマ絵が再生数を生む3つの心理的理由

「なぜ雑な絵の動画が伸びるのか？」

答えは心理学にある。

**理由1：「完璧でないもの」に人は引き寄せられる**

心理学では「不完全性効果」と呼ばれる現象がある。

完璧に整った情報より、少し欠けている情報の方が「続きを知りたい」という欲求が強まる。

ヘタウマ絵は視覚的に「続きが気になる」状態を作り出す。

「次の絵はどんな絵だ？」という軽い好奇心が、最後まで視聴させる力になる。

**理由2：企業っぽさがなく「信頼」される**

洗練された映像は「広告感」を醸し出す。

広告感が出た瞬間、人は防衛反応として離脱しやすくなる。

ヘタウマ絵には「個人が作った感」がある。

「この人が一人で作ったんだな」という親近感が、エンゲージメント（コメント・いいね）を生む。

**理由3：聴覚主導の「ながら視聴」に最適**

ヘタウマ絵×ナレーションの動画は、実は「ポッドキャストに近い体験」を提供している。

画面を見なくても、ナレーションだけで内容が理解できる。

通勤・家事・運動中の「ながら視聴」に対応できるため、1人あたりの視聴時間が伸びやすい。

YouTubeのアルゴリズムは「視聴維持率」を重視する。

ながら視聴で最後まで聴かれる動画は、アルゴリズムに評価される。

この3つが重なって、ヘタウマ絵チャンネルは「普通の動画より伸びやすい」という逆転現象が起きている。

## 画像切り替え2〜3秒の法則

「なぜ100枚の画像が必要なのか？」

10分の動画で100枚の画像を使うと、1枚あたりの表示時間が6秒になる。

さらに1枚を2〜3秒ずつ複数に分割すると、実質的な切り替えは2〜3秒ごとになる。

**2〜3秒ごとに新しい刺激が来ると、視聴者の注意は前に引っ張られ続ける。**

「退屈する前に次が来る」サイクルが、視聴維持率を押し上げる。

これがヘタウマ絵×大量画像の組み合わせが機能するメカニズムだ。

Claude Codeが「1回の命令で画像100枚を自動生成」できる理由も、このフォーマットに合わせてある。

英語圏でこのフォーマットが機能しているのに、日本語圏では同じ手法を実践しているチャンネルがほぼない。

理由は単純だ。

日本語の情報として公開されていないからだ。

先行者として動ける時間は、あと1〜2年かもしれない。

竸合が増える前にチャンネルを育て始めた人が、圧倒的に有利な位置を取れる。

これが「今すぐ始めるべき理由」だ。

## 月収61万円チャンネルの「再現可能性」を検証する

「本当に再現できるの？」

この疑問は正しい。

再現性を担保するために、参照チャンネルの構造をもう少し分解しておこう。

**動画の長さ：平均8〜12分**

YouTubeの広告単価は動画の長さに比例する部分がある。

8〜12分は「1本の動画に中間広告を入れられる最低ライン」だ。

画像100枚×2〜3秒の切り替えで、ちょうどこの尺に収まる。

**ナレーションのスタイル：読み上げ型**

難しい語りかけは必要ない。

スクリプトを書いて（あるいはAIで生成して）、それをそのまま読み上げる。

抑揚がなくても、ナレーションのリズムと絵の切り替えが組み合わさると、視聴者は「引き込まれる感覚」を覚える。

**アップロード頻度：月3〜5本からスタート**

参照チャンネルは1ヶ月で12本という驚異的なペースだったが、これを最初から目指す必要はない。

月3〜5本でも、6ヶ月続ければ18〜30本になる。

チャンネルが育つには「量と継続」が必要だ。

1本を完璧に仕上げるより、3本を量産する方が圧倒的に正しい。

## STEP 1: Claude Codeのセットアップ

### この段階での課題

Claude Codeは「Claude CLI」とも呼ばれるコマンドラインツールだ。

「コマンドライン」と聞いて怯んだ人もいるかもしれないが、心配しなくていい。

必要なコマンドはこの教材に全部書いてある。

コピペするだけで動く。

### 準備するもの

Claude Codeを使うには、**AnthropicのProプラン**への加入が必要だ。

月額は約3,000〜4,000円。

これが「唯一の固定費」であり、初期投資として必要なコストだ。

Proプランがない場合は、anthropic.comからサインアップして加入しておこう。

### インストール手順

**STEP 1-1: Node.jsをインストール**

Claude CodeはNode.js上で動く。

まずnodejs.orgにアクセスして「LTS（推奨版）」をダウンロード・インストールする。

Windowsならインストーラーをダブルクリックするだけだ。

インストール後、ターミナル（Windowsならコマンドプロンプト）を開いて以下を入力する。

```
node --version
```

バージョン番号が表示されれば成功だ。

**STEP 1-2: Claude Codeをインストール**

ターミナルで以下のコマンドを実行する。

```
npm install -g @anthropic-ai/claude-code
```

インストールが完了したら、以下で動作確認する。

```
claude --version
```

バージョン番号が表示されれば、Claude Codeのインストールは完了だ。

**STEP 1-3: AnthropicアカウントとAPIキーの連携**

初回起動時に認証が求められる。

```
claude
```

と入力すると、ブラウザが開いてAnthropicアカウントへのログインが求められる。

Proプランのアカウントでログインすると、Claude Codeが使えるようになる。

### ✅ POINT

Claude Codeの「Skills（スキル）機能」を使うと、よく使う作業をMarkdownファイルに手順として保存しておける。

今後の画像自動生成は、このスキル機能を使って繰り返し実行することになる。

## STEP 2: Hixfieldのセットアップ

### この段階での課題

Claude Codeが「指示を出す脳」なら、Hixfieldは「絵を描く手」だ。

Claude Codeからの指示を受け取って、ヘタウマ絵風の画像を自動生成する。

### 登録手順

**STEP 2-1: アカウント作成**

hixfield.comにアクセスして、メールアドレスでアカウントを作成する。

無料プランから始められる。

最初は無料枠の範囲でテストして、動作確認が取れたら有料プランへの移行を検討しよう。

**STEP 2-2: APIキーの取得**

ログイン後、ダッシュボードの「API」セクションから「Create API Key」をクリックする。

生成されたAPIキーをメモ帳などに控えておく。

このキーをClaude Codeに渡すことで、Claude Codeが「Hixfieldに画像生成を命令する」ことができるようになる。

**STEP 2-3: テスト画像を1枚生成する**

APIキーを取得したら、まずHixfieldのサイト上でテスト画像を1枚生成してみる。

プロンプト（指示文）に「a simple stick figure drawing of a person walking」と入力してみよう。

ヘタウマ風の人物画像が生成されれば、セットアップ完了だ。

### ✅ POINT：プロンプトのコツ

Hixfieldで「ヘタウマ感」を出すには、プロンプトに以下のような指定を入れると効果的だ。

「hand-drawn style」「rough sketch」「simple line drawing」「doodle art」

これらのキーワードをプロンプトに含めると、AIが「わざとへたな絵」を生成してくれる。

## STEP 3: Turboscribeのセットアップ

### この段階での課題

「タイムスタンプ付きの文字起こし」がなぜ必要なのか、疑問に思うかもしれない。

理由はシンプルだ。

ナレーション音声を「何秒の時点でどんな内容を話しているか」と紐付けることで、Claude Codeが「この言葉には画像Aを、次の言葉には画像Bを」と自動的に割り当てられるようになるからだ。

このタイムスタンプ情報がないと、画像とナレーションのタイミング合わせを手動で行う必要が生じる。

Turboscribeがこの作業を完全自動化してくれる。

### 登録手順

**STEP 3-1: アカウント作成**

turboscribe.aiにアクセスして、アカウントを作成する。

無料プランでも月数時間の文字起こしが可能だ。

**STEP 3-2: 音声ファイルをアップロードしてテスト**

手持ちの音声ファイル（スマホのボイスメモでも可）をアップロードしてみよう。

ファイルをドラッグ&ドロップするだけで文字起こしが始まる。

完了すると「タイムスタンプ付きのテキスト」が出力される。

たとえばこんな形式だ。

```
[00:00:03] こんにちは。今日はYouTubeチャンネルの始め方について
[00:00:08] お話しします。まず最初のステップは…
```

この形式でテキストが出力されれば、Turboscribeのセットアップは完了だ。

### ✅ POINT：文字起こしの精度

Turboscribeは日本語の音声にも対応しているが、雑音が多い環境で録音した音声は精度が下がる。

静かな場所でスマホのマイクに近づいて録音するだけで、精度は大きく改善する。

専用のマイクは不要だ。

## 3ツール連携の全体像

## STEP 1：スクリプトを用意する

### この段階での課題

「何を話せばいいかわからない」が最初の壁だ。

でも安心してほしい。

**スクリプトはAIに書いてもらえる。**

### 具体的なやり方

Claude Codeに以下のような形式で依頼する。

```
「[ニッチのテーマ]について、8〜10分で話せるナレーション用スクリプトを書いてください。
・ヘタウマ絵+ナレーション形式の動画用です
・1文を15〜25文字程度に短く区切ってください
・話し言葉で書いてください
・画像の切り替えポイントに（切り替え）と入れてください」
```

たとえばニッチが「猫の不思議な行動」なら、こう依頼する。

```
「猫が高い場所から落ちても怪我しない理由について、8〜10分で話せるナレーション用スクリプトを書いてください…」
```

2〜3分でスクリプトが完成する。

内容を確認して「これでいい」と思えたら次のステップへ進む。

内容が気に入らなければ「もう少し面白い導入にして」と追加指示するだけだ。

### ✅ POINT：スクリプトの長さの目安

8〜10分の動画に必要な文字数は約2,500〜3,500字だ。

Claudeに「2,500字程度のスクリプトを書いて」と指定すると時間の計算がしやすい。

## STEP 2：ナレーションを録音する

### この段階での課題

「声が良くない」「滑舌が悪い」という不安が出やすいステップだ。

でもヘタウマ絵のチャンネルでは、ナレーションの「完璧さ」は求められていない。

自然な話し言葉で、普通のスピードで読み上げるだけでいい。

### 具体的なやり方

**方法A：自分で録音する（推奨）**

スマホの標準ボイスレコーダーアプリで十分だ。

静かな部屋で、マイクに口を近づけて（10〜20cm程度）スクリプトを読み上げる。

NG部分は後で切れるので、途中で詰まっても止めずに続けていい。

録音が終わったら、PCに転送してmp3またはm4a形式で保存する。

**方法B：Fiverr（ファイバー）で外注する**

英語ナレーターを使う場合はFiverrが便利だ。

「voiceover artist」で検索すると、1本あたり5〜20ドル程度でプロが録音してくれる。

日本語ナレーターも探せばいる（「Japanese voiceover」で検索）。

**費用の目安：** 自分で録れば無料 / Fiverr外注なら1本5〜20ドル

## STEP 3：Turboscribeでタイムスタンプ付き文字起こし

### この段階での課題

ナレーション音声をそのままClaude Codeに渡しても、「何秒の時点で何を話しているか」がわからない。

Turboscribeでタイムスタンプ付きテキストに変換することで、Claude Codeが「このセリフのタイミングに画像Aを配置する」という指示を正確に実行できるようになる。

### 具体的なやり方

**STEP 3-1:** turboscribe.aiにログインして、録音したmp3ファイルをアップロードする。

**STEP 3-2:** 言語設定を「日本語」に変更する。

**STEP 3-3:** 「Transcribe」ボタンを押す。

**STEP 3-4:** 2〜5分で文字起こしが完了する。

**STEP 3-5:** 「タイムスタンプ付き」形式でエクスポートする（テキストファイルまたはSRTファイル）。

以下のような形式で出力される。

```
[00:00:02] 今日は猫が高いところから落ちても
[00:00:05] 怪我しない理由について話します。
[00:00:09] これは「ライティング反射」と呼ばれる能力で…
```

このテキストファイルをそのままClaude Codeに渡す。

## STEP 4-5：Claude Codeに投げて画像100枚を自動生成する

### この段階での課題

ここが「AIで動画を作る」というメソッドの核心だ。

Claude Codeにタイムスタンプ付きテキストを渡し、「各セリフの内容に合った画像をHixfieldで100枚生成して、ファイル名をタイムスタンプにして保存する」という一連の作業を自動化する。

### 具体的なやり方

**STEP 4-1:** 新しいフォルダを作成する。名前は「video_001」など。

**STEP 4-2:** Turboscribeで出力したタイムスタンプ付きテキストファイルを、このフォルダに入れる。

**STEP 4-3:** Claude Codeのターミナルで、このフォルダに移動する。

```
cd ~/Desktop/video_001
```

**STEP 4-4:** Claude Codeに以下のような指示を出す。

```
「script.txtのタイムスタンプ付きテキストを読んで、各セリフの内容に合ったヘタウマ絵スタイルの
画像をHixfield APIで生成してください。
・画像スタイル: hand-drawn doodle style, simple line drawing
・ファイル名: タイムスタンプをそのまま使う（例: 000002.png, 000005.png）
・保存先: imagesフォルダ
・合計枚数: 最低100枚になるように調整する」
```

**STEP 4-5:** Claude Codeが自動的にHixfield APIを呼び出して、100枚の画像を生成・保存していく。

この処理は5〜15分かかる（生成枚数と画像サービスの速度による）。

その間、あなたは待っているだけでいい。

---

## STEP 6：ファイル名＝タイムスタンプで自動保存の仕組み

ここが「手動作業ゼロ化」の核心だ。

画像ファイルのファイル名が「タイムスタンプ（秒数）」になっているため、動画編集ソフトが自動的に「正しい順番」で並べることができる。

たとえば：
- 000002.png → 2秒の時点で表示
- 000005.png → 5秒の時点で表示
- 000009.png → 9秒の時点で表示

このファイル命名規則により、「手動でどの画像をどのタイミングで使うか決める」という作業が完全になくなる。

Claude Codeがこの命名規則を自動で実行するように指示に含めるのがポイントだ。

## STEP 7：動画編集ソフトに並べるだけ（10分で完了）

### この段階での課題

「動画編集は難しい」という先入観があるかもしれない。

でもこのメソッドでは、編集ソフトを使う作業は1つだけだ。

**「ファイル名順に並んだ画像100枚を、タイムラインに並べる」**

それだけだ。

### 具体的なやり方

無料で使える動画編集ソフトとして「DaVinci Resolve（ダヴィンチリゾルブ）」をおすすめする。

プロの映像制作者も使う本格的なソフトだが、今回の用途では数分で操作を覚えられる。

**STEP 7-1:** DaVinci Resolveを起動して、新規プロジェクトを作成する。

**STEP 7-2:** imagesフォルダの中身を全選択して、タイムラインにドラッグ&ドロップする。

ファイル名の順番（タイムスタンプ順）で自動的に並ぶ。

**STEP 7-3:** ナレーション音声（mp3）を音声トラックに追加する。

**STEP 7-4:** 画像の表示時間を一括調整する（全選択→右クリック→「クリップ時間を変更」→2〜3秒に設定）。

**STEP 7-5:** 書き出し（エクスポート）をクリックして、mp4形式で保存する。

これで動画1本が完成だ。

かかる時間：10〜15分。

初回は不慣れで少し時間がかかるかもしれないが、2回目以降は10分以内になる。

## よくある質問・つまずきポイント

**Q: Claude Codeがうまく動かない場合はどうすれば？**

A: 最も多いのは「APIキーの設定ミス」だ。HixfieldのAPIキーがClaude Codeの設定ファイルに正しく書き込まれているか確認しよう。設定ファイルのパスは環境によって異なるが、Claude Code起動時に表示されるログに確認方法が出ている。

**Q: 生成された画像がスクリプトの内容と全然合っていない場合は？**

A: Hixfieldへのプロンプトの指示を修正する。「より具体的な描写」を加えると精度が上がる。たとえば「cat falling from height」ではなく「simple stick figure drawing of a cat with its legs spread out falling from a tall tree」のように詳細化する。

**Q: タイムスタンプの数が100枚に足りない場合は？**

A: Claude Codeへの指示に「タイムスタンプが少ない場合は、同一タイムスタンプ内で複数枚生成して合計100枚にする」と追加指示する。あるいはTurboscribeの出力設定で「より細かいタイムスタンプ間隔」を選ぶことで解決できる。

**Q: DaVinci Resolveのどのバージョンを使えばいいか？**

A: 無料版（DaVinci Resolve 19）で十分だ。有料版のStudioは不要。ダウンロードはblackmagicdesign.comから無料で入手できる。

# 第5章：11Labsを使ってはいけない理由とリスク管理

「なぜ11Labsの話をわざわざするの？」

そう思うかもしれない。

でもこの章は、正直言って「最も大切な章」のひとつだ。

なぜなら、多くのYouTube副業解説記事が**11Labsをおすすめしているから**だ。

「AI音声生成なら11Labs」という流れが副業界隈で定着しつつある中で、11Labsを使ったチャンネルが収益化停止になっているケースが出始めている。

せっかく動画を量産してチャンネルを育てても、収益化が止まってしまっては意味がない。

この章を読んでおくことで、そのリスクを完全に回避できる。

---

## 11Labsとは何か、なぜリスクがあるのか

**11Labs（ElevenLabs）** は高品質なAI音声生成サービスだ。

テキストを入力すると、人間そっくりの自然な音声を生成してくれる。

声の種類も豊富で、しかも日本語にも対応している。

一見すると「最高のツール」に見える。

だがここに落とし穴がある。

**YouTubeの収益化ポリシーとAI音声の関係だ。**

YouTubeは2023年以降、「AI生成コンテンツ」に関するポリシーを強化している。

特に「AIが生成した音声（ナレーション）のみで構成された動画」は、コンテンツの独自性が低いとみなされる可能性がある。

実際に、AI音声を使ったチャンネルで収益化停止や登録削除の報告が増えている。

問題になりやすいパターンはこうだ。

・AI音声のみでナレーションを構成している
・スクリプトもAI生成、音声もAI生成、画像もAI生成という「完全AI自動化」
・同じ声のAI音声を複数チャンネルで使い回している

**11Labsを使うこと自体が直接の規約違反ではないが、「完全AI自動化」という属性がYouTubeのアルゴリズムに検出されやすくなっている。**

---

## 収益化停止リスクの実態

「収益化停止」がどれほど痛いか、具体的に考えてみよう。

100本の動画を積み上げて、月10万円の収益が見えてきた頃に突然「収益化の資格を喪失しました」という通知が来たとしたら？

動画は残っているが、収益はゼロに戻る。

再審査申請は可能だが、通過の保証はない。

これが現実のリスクだ。

100本分の作業が一瞬で無効になるリスクを背負って走るより、最初から安全なルートを選ぶ方が合理的だ。

---

## 安全な代替案：2つの選択肢

**選択肢A：自分の声で録音する（最も推奨）**

これが最も安全で、かつ「チャンネルの個性」にもなる方法だ。

自分の声には「人間らしさ」がある。

抑揚・間・感情がAI音声にはない形で乗る。

視聴者は「この人の声で聴きたい」という感覚を持ちやすく、ファンが生まれやすい。

最初は「自分の声が嫌い」「滑舌が気になる」という抵抗があるかもしれない。

でも実際のヘタウマ絵チャンネルの視聴者は、ナレーターの声の質より「内容」を求めている。

普通に読み上げるだけで十分だ。

**選択肢B：Fiverr（ファイバー）で人間のナレーターを外注する**

予算があれば、1本5〜20ドルで人間のナレーターに依頼できる。

Fiverrで「Japanese voiceover」で検索すると、日本語ネイティブのナレーターが見つかる。

1本あたりのコストは増えるが、チャンネルの収益化リスクを下げられる。

収益化後に売上の一部を外注費に回すという使い方もある。

## 収益化の条件と現実的タイムライン

YouTube収益化（YPP: YouTube Partner Program）の加入条件を確認しておこう。

**YPP加入条件（2026年現在）**

登録者数：1,000人以上
総再生時間：過去12ヶ月で4,000時間以上
または ショート動画の視聴回数：1,000万回以上（ショート専用）

**現実的なタイムライン（月10本ペースの場合）**

スタート〜3ヶ月：30本の動画公開。チャンネルの認知度がゼロから始まる。再生数は少ないが動画資産が積み上がっていく。

3〜6ヶ月：動画本数が30〜60本になる頃から、アルゴリズムが動画を拾い始める。再生数が徐々に増え始める。

6〜12ヶ月：ニッチが正しければ、この時期に収益化基準を達成するチャンネルが出てくる。

**ただし、ニッチ選定を間違えると全部無駄になる。**

これが最大のリスクだ。

## ニッチ選定の方法：VidIQを使う

ニッチ選定の具体的な手順を解説する。

使うツールは**VidIQ**だ。

VidIQはYouTube上のキーワード需要・競合チャンネルの分析ができる無料ツールだ。

Chrome拡張機能として追加すると、YouTube検索時に「このキーワードの検索ボリューム」と「競合の強さ」が表示される。

**ニッチ選定のポイント3つ**

1. 検索ボリュームが「中程度以上」あること（完全な無名ジャンルは需要がない）
2. 英語圏では動画があるが日本語圏にはほとんどない（翻訳タイムラグを活用）
3. 継続的に動画ネタが出てくるジャンルであること（ネタ切れを防ぐ）

**ニッチ選定の手順**

STEP 1：英語でYouTube検索して「ヘタウマ絵+ナレーション」形式の動画が伸びているジャンルを探す（例：「animal facts doodle」「history explained doodle」など）

STEP 2：VidIQで同じキーワードの日本語版を検索して、検索ボリュームを確認する

STEP 3：日本語で同じジャンルの競合チャンネルがほとんどなければ、参入のチャンスだ
