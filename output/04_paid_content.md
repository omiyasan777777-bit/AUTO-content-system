## なぜ他の方法ではなくa-Shell×Pythonなのか

「iPhoneからアメブロを自動投稿する方法」は、実は他にもいくつかあります。

例えば、Zapier（ザピアー）などの自動化サービスを使う方法。
または、ショートカットアプリを使う方法。

でも、これらの方法には共通した限界があります。

**「できることがあらかじめ決まっている」** という限界です。

Zapierはテンプレートの中でしか動きません。
ショートカットアプリもiOSの機能の範囲内でしか動きません。

自分のやりたいことが少し特殊だったり、
カスタマイズしたくなったときに、すぐ壁にぶつかります。

一方、a-Shell×Pythonには限界がありません。

コードは自分で書くものなので、
「やりたいこと」があればそれをコードで表現できます。

カテゴリを自動で分類したい。
タグを自動で付けたい。
投稿時間を毎日ランダムにしたい。

すべて、Pythonのコードで実現できます。

もちろん最初は「コピペ」でOKです。

でも、この教材でコードの意味を理解すれば、
いつかは自分でカスタマイズできる力が身に付きます。

それが、a-Shell×Pythonを選ぶ理由です。

## STEP1：AtomPub APIとは何かを理解する

### この段階での課題

「APIという言葉の意味がわからず、何をしようとしているのか頭に入ってこない」

### 解決策

APIをひとことで言うと「外部のプログラムからブログを操作する窓口」です。
アメブロが公式に用意している機能なので、安心して使えます。
難しい知識は必要ありません。

これができると...

- 自分が何をしているのかの全体像が見える
- APIキー取得の作業の「目的」が明確になる
- スクリプト作成時にコードの意味が理解しやすくなる

### AtomPubとは

AtomPub（Atom Publishing Protocol）は、ブログ記事をプログラムから投稿・管理するための国際規格です。

「国際規格」と聞くと難しそうですが、
要するに「こういう形式でデータを送ったら記事を投稿できる」というルールです。

アメブロはこのAtomPubに対応しているため、
正しい形式でデータを送れば、プログラムから記事を投稿できます。

今回は「requests」ライブラリを使って、
このルールに従ったデータを送るスクリプトを作ります。

### APIキーとは

APIキーとは「あなたがアメブロのアカウント所有者であることを証明するパスワード」です。

自分のアカウントに記事を投稿するためには、
「このブログの持ち主です」という証明が必要です。

アメブロでは、管理画面からAPIキーを発行できます。

このキーをスクリプトに組み込むことで、
「あなたのブログに投稿する権限がある」とアメブロのサーバーが認識してくれます。

### ✅ POINT

APIキーは他人に知られると、あなたのブログに勝手に記事を投稿されるリスクがあります。スクリプトファイルに書き込む際は、他人に見せないようにしましょう。

## STEP3：APIキーとブログIDを控える

### この段階での課題

「APIキーってどこに書いてあるの？どれをコピーすればいいの？」

### 解決策

AtomPubを有効化すると、設定画面に必要な情報が表示されます。
3つの情報を控えるだけです。

これができると...

- スクリプトに組み込むべき認証情報がそろう
- 次の章でコードを書くときにすぐ使える
- 接続テストができるようになる

### 控えるべき3つの情報

AtomPubを有効化した後、設定画面に以下の3つが表示されます。

**① アメブロID（Ameba ID）**
あなたのアメブロのログインIDです。
例: `your-ameba-id`

**② APIパスワード（AtomPub用のパスワード）**
AtomPub専用のパスワードです。
通常のログインパスワードとは別物です。
例: `xxxxxxxxxxxxxxxx`

**③ ブログID（投稿先のブログURL）**
あなたのアメブロのURLの一部です。
例: `ameblo.jp/your-blog-id/` の「your-blog-id」の部分

### 具体的なやり方

## STEP2：投稿に必要なXML形式を理解する

### この段階での課題

「XMLって何？HTMLと違うの？」

### 解決策

XMLはHTMLに似た「タグ」を使うデータ形式です。
アメブロに投稿する際に使うXMLのひな型を先に見ておくと、次のコードが理解しやすくなります。

これができると...

- XMLの構造に慣れる
- コードの「どの部分がタイトルで、どの部分が本文か」がわかる
- 自分でカスタマイズするときに変更すべき箇所がわかる

### アメブロAPIが受け取るXMLの形

アメブロの投稿APIに送るXMLは、こんな形をしています。

```xml
<?xml version="1.0" encoding="utf-8"?>
<entry xmlns="http://www.w3.org/2005/Atom"
       xmlns:app="http://www.w3.org/2007/app">
  <title>記事のタイトル</title>
  <content type="text/plain">記事の本文</content>
  <app:control>
    <app:draft>no</app:draft>
  </app:control>
</entry>
```

各タグの意味を解説します。

**`<title>`タグ**
→ 記事のタイトルを入れる場所

**`<content type="text/plain">`タグ**
→ 記事の本文を入れる場所。`type="text/plain"`はプレーンテキストで送ることを意味します

**`<app:draft>no</app:draft>`タグ**
→ 「no」で即時公開、「yes」で下書き保存になります

このXMLを文字列としてPythonで作り、アメブロのAPIに送信することが基本の流れです。

### ✅ POINT

`<content type="text/html">`に変更すると、HTMLタグを含む記事本文を送ることができます。太字や改行タグを使いたい場合に活用してください。

- 原因を特定して素早く修正できる
- 次回から同じエラーで詰まらなくなる

### エラー対処法一覧

**エラー①：ステータスコード 401（Unauthorized）**

原因: 認証情報が間違っている

対処法:
- AMEBA_IDが正しいか確認する（アメブロのログインIDと同じです）
- API_PASSWORDが正しいか確認する（通常のログインパスワードではなく、AtomPub設定画面に表示されるAPIパスワードです）
- コピー時にスペースが混入していないか確認する（IDやパスワードの前後に空白があるとエラーになります）

```python
# 確認用：コードの中の値を出力して確認する
print(f"ID: [{AMEBA_ID}]")
print(f"PASSWORD: [{API_PASSWORD}]")
print(f"BLOG_ID: [{BLOG_ID}]")
```

`[]` の中に余分なスペースがないか目視で確認しましょう。

---

**エラー②：ステータスコード 403（Forbidden）**

原因: AtomPub機能が有効化されていない

対処法:
- アメブロの管理画面 → ブログ設定 → 外部サービス連携 → AtomPubが有効になっているか確認する
- 一度設定を保存し直してみる

---

**エラー③：ステータスコード 404（Not Found）**

原因: URLが間違っている（ブログIDが間違っている）

対処法:
- BLOG_IDが正しいか確認する
- アメブロのURLが `https://ameblo.jp/your-blog-id/` であれば、BLOG_IDは `your-blog-id` の部分

---

**エラー④：requests.exceptions.ConnectionError**

原因: インターネットに接続されていない

対処法:
- iPhoneのWi-Fiまたはモバイルデータ通信がオンになっているか確認する
- 電波の弱い場所では接続エラーが出ることがあります。場所を変えて再試行してください

---

**エラー⑤：UnicodeDecodeError / 文字化け**

原因: 文字コードの問題

対処法:
- `data=xml_data.encode("utf-8")` の行が正しく書かれているか確認する
- `.encode("utf-8")` がない場合、日本語の文字化けやエラーが起きます

---

**エラー⑥：SyntaxError（構文エラー）**

原因: コードの書き方が間違っている

対処法:
- エラーメッセージに「File "post_to_ameblo.py", line ○○」と書かれています
- その行番号のコードを確認して、スペルミスや閉じ忘れ（`"`や`}` の欠落）を探してください
- a-ShellのnanoエディタでCtrl+_ を押すと指定した行番号に飛べます

### ✅ POINT

エラーメッセージは英語ですが、一番大事なのは「ステータスコード」（401・403・404などの数字）と「最後の行のエラー文（SyntaxError等）」の2点です。この2点を確認するだけで、ほとんどのエラーの原因が特定できます。

## STEP3：応用カスタマイズのアイデア

### この段階での課題

「基本の投稿はできた。でも、もっと便利にしたい」

### 解決策

スクリプトを少し変えるだけで、様々な機能が追加できます。
コードの意味がわかっているあなたなら、今すぐ試せるアイデアを紹介します。

これができると...

- 自動化の幅が広がり、より手間が省ける
- 自分だけのカスタムスクリプトが作れる
- 「Pythonで何でもできる」という手応えを感じられる

---

**カスタマイズ①：下書き保存モードを追加する**

投稿前に内容を確認したい場合は、下書きで保存できます。

```python
# app:draftをyesにするだけ
xml_data = f"""...
  <app:control>
    <app:draft>yes</app:draft>
  </app:control>
...</code>"""
```

`no` を `yes` に変えるだけで下書き保存になります。

---

**カスタマイズ②：テキストファイルから本文を読み込む**

毎回スクリプトの中身を編集するのが面倒な場合は、
テキストファイルを読み込む方式にできます。

```python
# 本文をテキストファイルから読み込む
with open("article.txt", "r", encoding="utf-8") as f:
    BODY = f.read()
```

`article.txt` というファイルを作って本文を書いておけば、
スクリプトを変更せずに記事の内容を変えられます。

---

**カスタマイズ③：投稿日時を引数で指定する**

コマンドラインから実行時に記事タイトルを渡せるようにできます。

```python
import sys

# コマンドライン引数からタイトルを取得
if len(sys.argv) > 1:
    TITLE = sys.argv[1]
else:
    TITLE = "デフォルトタイトル"
```

実行時に `python3 post_to_ameblo.py "今日の記事タイトル"` と打つだけで、タイトルを動的に変えられます。

---

**カスタマイズ④：カテゴリを指定する**

アメブロのAtomPubではカテゴリ（テーマ）の指定もできます。

```python
CATEGORY = "日常"  # 投稿したいカテゴリ名

xml_data = f"""...
<entry ...>
  <title>{TITLE}</title>
  <content ...>{BODY}</content>
  <category term="{CATEGORY}" />
  <app:control>
    <app:draft>no</app:draft>
  </app:control>
</entry>"""
```

`<category term="カテゴリ名" />` を追加するだけです。

---

**カスタマイズ⑤：ショートカットアプリと連携する**

iPhoneのショートカットアプリから a-Shellのスクリプトを実行できます。

設定方法:
1. ショートカットアプリを開く
2. 「新規ショートカット」を作成
3. 「a-Shell」の「Runコマンド」アクションを追加
4. コマンドに `python3 post_to_ameblo.py` を入力
5. ショートカットをホーム画面に追加する

これにより、ホーム画面のアイコンをタップするだけで投稿スクリプトが実行されます。

ウィジェットに追加すれば、ロック画面からワンスワイプで実行することも可能です。

### ✅ POINT

カスタマイズは「1つずつ試す」のが基本です。複数の変更を一度に加えると、どの変更が原因でエラーが出たかわからなくなります。1つ変えたら動作確認、次の変更、という順序で進めましょう。
