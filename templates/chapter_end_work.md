# 各章末ワーク テンプレート集

> 読んですぐ自分の発信に適用できる実践形式。
> 各章の最後にこのテンプレートから1つ以上選んで使用する。

---

## 使い方

1. 章の内容に合ったワーク型を選ぶ
2. `{　}` の部分をその章のテーマに合わせて書き換える
3. 読者が「手を動かせる」具体的なアクションに落とし込む

---

## ワーク型一覧

### 型A：穴埋め式ワーク（最短・最も取り組みやすい）

```
📝 章末ワーク

この章の内容を、あなたの発信に当てはめてみましょう。

以下の文章を完成させてください。

---

私のターゲットは「{　　　　}」という悩みを持つ人です。

この章で学んだ「{章のメインノウハウ}」を使うと、
私のターゲットに向けて「{　　　　}」を伝えられます。

実際に使う場面は「{　　　　}」です（例：Twitter投稿 / note の見出し / LINEメッセージなど）。

---

✅ 書けたら、実際にその媒体に投稿 or 下書き保存してみてください。
```

---

### 型B：3ステップ実践ワーク（ノウハウ章向け）

```
📝 章末ワーク ── 3ステップで今すぐ使う

STEP 1｜自分のテーマに置き換える
この章で解説した「{章のノウハウ名}」を、
あなたのジャンルに当てはめると？

→ 私のジャンルでは「{　　　　}」になる。

---

STEP 2｜1つ試しに作ってみる
STEP 1で気づいた内容で、以下を1つ書いてみましょう。

（例）
・投稿の冒頭1文
・見出し候補 3本
・CTAの文章

→ 書いた内容: 「{　　　　}」

---

STEP 3｜発信する or ストックする
書けたものを、すぐに投稿 or メモアプリに保存してください。
ためらわず、まず出す。完成度より数です。

✅ ここまでやって、ようやく「章の内容を使った」状態になります。
```

---

### 型C：チェックリスト型ワーク（確認・振り返り向け）

```
📝 章末チェックリスト

この章の内容が身についているか確認しましょう。

□ {チェック項目 1：章の核心概念を自分の言葉で説明できる}
□ {チェック項目 2：自分のジャンルへの応用例が1つ言える}
□ {チェック項目 3：今日中に試せるアクションが決まっている}
□ {チェック項目 4：この手法を使わないと損すること・リスクがわかっている}
□ {チェック項目 5：次の章でさらに深掘りされる概念と繋がっている}

---

チェックが3つ以下の方へ:
この章をもう一度読み直してから次に進んでください。
理解より実践。読んだだけでは何も変わりません。
```

---

### 型D：比較ビフォーアフター記入ワーク（変化の実感向け）

```
📝 章末ワーク ── ビフォー → アフター

この章を読む「前」と「後」で、あなたの認識はどう変わりましたか？

| | 内容 |
|---|---|
| **Before（読む前の理解）** | {　　　　} |
| **After（読んだ後の理解）** | {　　　　} |
| **明日から変えること** | {　　　　} |

---

「Before → After」が埋まらない場合は、
この章で「へえ」と思った一文を抜き書きするだけでもOKです。

→ 印象に残った一文: 「{　　　　}」

✅ この気づきをSNSに投稿すると、発信ネタにもなります。
```

---

### 型E：アウトプット投稿型ワーク（SNS連動向け）

```
📝 章末ワーク ── 今すぐ投稿できるコンテンツを作る

この章の内容を使って、SNS投稿を1本作りましょう。

▼ 以下のフォーマットに沿って書いてください

---

【投稿タイトル（フック）】
{　　　　}

【本文（気づき・ノウハウ・体験談）】
{　　　　}

【締め（CTA or まとめ）】
{　　　　}

---

文字数目安: 140〜300字（Twitter/X） or 300〜600字（note見出し）

✅ 書けたら実際に投稿しましょう。
「わかった」ではなく「やった」に変えることが大事です。
```

## 章ごとのワーク選択ガイド

| 章の性質 | おすすめワーク型 |
|---|---|
| 概念・考え方を学ぶ章 | 型A（穴埋め）or 型D（ビフォーアフター） |
| 手順・スキルを学ぶ章 | 型B（3ステップ実践） |
| 総まとめ・振り返り章 | 型C（チェックリスト） |
| SNS運用・発信系の章 | 型E（アウトプット投稿） |
| 全章共通 | どれでも組み合わせ可 |

---

## 注意点

- ワークは**読者が5〜10分で完了できる量**に絞る
- 「正解を書く」のではなく「あなた自身の言葉で埋める」形にする
- 完成形の例示（記入例）を1つ添えると取り組みやすくなる
- 各章に1つ以上のワークを必ず入れること（`04_paid_content.md` のルールに準拠）
